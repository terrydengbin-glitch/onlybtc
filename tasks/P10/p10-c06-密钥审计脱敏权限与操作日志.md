# P10-C06 密钥审计、脱敏、权限与操作日志

## 状态

DONE

## 所属 Phase

P10 API Key 配置与密钥管理

## 任务目标

保证 API Key 的显示、保存、测试和清空过程可审计、可脱敏、可控制权限。

## 实施范围

- 密钥脱敏工具。
- 操作审计日志：created/updated/tested/deleted。
- 不记录明文 key。
- 配置接口预留权限校验。
- 错误日志脱敏。

## 验收标准

- 日志中搜索不到完整 key。
- 前端响应不含完整 key。
- 清空/更新操作有审计记录。

## 执行记录

- 新增 `backend/src/onlybtc/core/settings_audit.py`。
- 新增审计契约 `p10.c06.settings_key_audit.v1`，使用 `logs/settings-key-audit.jsonl` 保存脱敏操作日志。
- 新增 `GET /api/settings/audit`。
- `/api/settings` 增加 `settings_audit` summary。
- `POST /api/settings/env` 成功/失败都会记录审计事件：
  - `action=env_update`
  - `action=env_update_rejected`
  - 仅记录 env key、provider id、备份路径、operation counts，不记录 value。
- `POST /api/settings/providers/{provider_id}/test` 与 `test-all` 记录 `tested` 审计事件。
- `.env` 写入结果增加：
  - `created_keys`
  - `replaced_keys`
  - `deleted_keys`
  - `operation_counts`
- 加强日志脱敏，覆盖 `Authorization: Bearer ...`、`authorization=Bearer ...`、`api_key=...` 等格式。
- 前端 Settings / System 页面增加 Recent Key Audit 与 audit log 状态。
- 权限控制本轮保留 `actor=local_api` 与 guardrail `permission_gate_reserved_for_operator_auth`，为后续登录/操作者权限接入预留。
