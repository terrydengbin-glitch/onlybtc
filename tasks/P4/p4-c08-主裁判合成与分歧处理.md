## 2026-05-21 全链条对齐补充

本卡必须对齐 P4-C12。主裁判必须合成规则基线与多模型观点：

- 输入包含 pre-LLM baseline、model votes、challenges、P3 invalidations、adversarial flags。
- 多模型不得简单投票，必须按 evidence support、数据质量、分歧等级与状态机约束合成。
- 输出 `confidence_discount`、minority objection、accepted/rejected rationale。
- 写入 `judge_syntheses`，并绑定 `debate_id` 与 controller `run_id`。
- 不允许裁判覆盖 hard constraint，例如 run_mode critical block。

## 2026-05-21 Agent 化重构补充

Judge Agent 是 P4 的主裁判，但不是自由预测模型。它只能合成以下输入：

- P4-C03 pre-LLM rule baseline。
- P4-C04 状态机结果与 hard constraints。
- P4-C06 四个 Analyst Agent 输出。
- P4-C07 交叉质询与 revision。
- P3 alerts、event windows、invalidations。
- P1/P2/P8 数据质量、source conflict、fallback、run_mode integrity。

Judge Agent 输出必须包含 `judge_synthesis_id / debate_id / pack_id / controller_run_id / dominant_regime / trend_state / risk_state / consensus_level / disagreement_level / accepted_claims / rejected_claims / minority_objections / confidence / confidence_discount / blocked_by / publish_allowed / evidence_ids / state_machine_constraints_applied`。

裁判合成原则：不简单投票；优先接受 evidence support 更强、数据质量更高、且未违反 P3/P4 hard constraints 的观点；高分歧本身必须降低 confidence；少数派强反证必须保留；若反方审查或状态机阻断，Judge Agent 只能输出受限发布或禁止发布。
