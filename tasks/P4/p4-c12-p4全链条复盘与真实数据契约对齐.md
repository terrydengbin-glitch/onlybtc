## P4 新链路目标

P4 必须从“LLM 总结器”改为“受约束的总控融合系统”：

```text
P1 data quality + source health
  -> P2 radar module JSON
  -> P3 alerts / invalidations / event windows
  -> Evidence Pack freeze
  -> rule baseline / state machine
  -> 4 Analyst Agents
  -> Cross-examination Agent
  -> Judge Agent synthesis
  -> adversarial review
  -> final controller JSON
  -> dashboard snapshot / history replay
```

2026-05-21 起，P4 采用“自建 P4 Agent 业务编排 + OpenAI Agents SDK 执行层”的混合架构：

- onlyBTC 自建业务编排：Evidence Pack、4 分析师分工、规则基线、状态机、交叉质询、主裁判、反方审查、SQLite 落库和审计 HTML。
- OpenAI Agents SDK 只作为 runtime adapter：负责模型调用、structured output、tracing、guardrails、handoff/as_tool 等执行能力。
- 模型供应商不等同于业务角色；DeepSeek/Qwen/Kimi/OpenAI 之类属于 runtime 配置，`macro_event_analyst` 等才是业务角色。

## 需要同步的 P4 任务

### P4-C01 Analyst Agent 输入输出 Schema

必须定义真实 module input schema，读取 `module_json_outputs`、P3 anomaly/divergence/event/invalidation refs，不允许自由摘要。

### P4-C02 4 分析师 Agent Prompt 与证据约束

Prompt 必须引用 `evidence_id`、`metric_id`、`source_id`、`run_id`、`reason_code`，禁止输出无证据判断。

### P4-C03 规则权重融合引擎

融合必须先由规则层读取 P2/P3 结构化输出，生成 pre-LLM controller baseline。

### P4-C04 BTC 状态机与状态切换规则

状态机必须消费 P3 alert、event window、invalidation、run_mode integrity，不允许 LLM 越过状态机强行切换。

### P4-C05 Evidence Pack 生成器

Evidence Pack 必须冻结同一 run 的 P1/P2/P3/P8 证据，写入 `evidence_packs`、`evidence_items`、`evidence_metric_links`。
同时必须对齐 P4-C13：冻结全部 14 个 Radar module 的 `radar_outputs`
和 `module_json_outputs.features[]`，并把每个 feature 转换为可追溯
evidence item 或明确记录为 `provider_required / missing / suppressed`。

### P4-C06 4 分析师 Agent 独立分析执行器

独立盲审输入必须是 frozen Evidence Pack，不允许模型访问未冻结的最新数据。

### P4-C07 Analyst Agent 交叉质询与修正

质询必须围绕 evidence_id、invalidation reason、source conflict、data quality discount 展开。

### P4-C08 Judge Agent 主裁判合成与分歧处理

主裁判必须把分歧映射为 confidence discount、minority objection、publish constraints。

### P4-C09 反方审查机制

反方审查必须验证：是否忽略 P3 反证、是否绕过 run_mode critical block、是否输出交易建议。

### P4-C10 最终总控 JSON 与观察建议输出

最终 JSON 必须可写入 `judge_syntheses` 和 `dashboard_snapshots`，并被 P9/P5 直接消费。

### P4-C11 LLM 与总控 Mock 与 DoD 验收

Mock 不再使用孤立 fixture，必须由 P8 seed 或 P1/P2/P3 mock/full-chain run 生成。

### P4-C15 OpenAI Agents SDK Runtime Adapter

新增 runtime adapter 任务卡。P4 业务编排保持自建，OpenAI Agents SDK 负责执行层的 structured output、tracing 与 guardrails。所有 Agent 调用必须返回 `agent_run_id / trace_id / prompt_version / schema_version / guardrail_results`，并能写入 P8 `llm_*` 表。

### P4-C16 P4 Agent 全链条重跑与审计 HTML

新增 P4 独立全链条审计卡。P4 必须像 P1/P2/P3 一样生成本 run 的审计 HTML，展示 Evidence Pack coverage、4 个 Analyst Agent 输出、交叉质询、Judge 合成、反方审查和最终总控 JSON。

  - radar modules consumed count，必须为 14/14
  - radar feature items consumed count
  - signed event metrics consumed count，必须为 4/4
