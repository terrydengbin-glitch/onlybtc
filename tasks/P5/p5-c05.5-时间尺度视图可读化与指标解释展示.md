# P5-C05.5 时间尺度视图可读化与指标解释展示

## 状态

DONE

## 当前架构对齐（2026-05-22）

本任务只优化 P5 Vue3 展示层，不回改 P1/P2/P3/P4.5 契约。数据来源继续使用 P4.5 Research Report v2 的 `horizon_views`，包括 `direction`、`strength`、`confidence`、`support_drivers`、`pressure_drivers`、`dominant_side`、`dominant_drivers`、`interpretation`、`watch_rules`。

## 所属 Phase

P5

## 任务目标

将 Dashboard 的 24h / 3d / 7d 时间尺度视图从简短格式升级为“可直接阅读的文字 + 关键指标”展示。首页保留紧凑版，Overview 子页展示更完整版本。

## 实施范围

- 首页时间尺度卡片展示方向、强度、置信度、支撑驱动、压力驱动和观察规则摘要。
- Overview 子页展示完整 24h / 3d / 7d 卡片，接近审计 HTML 原报告的信息密度。
- 增加前端指标名映射，例如 `btc_1h_volume` 展示为 `1h 成交量`，但保留原 metric id 作为 title 或辅助信息。
- 仅使用已有 `horizon_views` 字段，不新增上游字段。
- 避免 raw JSON、Python dict、run_id 或内部 schema 污染主阅读区。
- 遵守观察与审计边界，不输出交易执行建议。

## 输入

- P4.5 `GET /api/p45/dashboard/latest`
- `horizon_views.h24 / d3 / d7`

## 输出

- Vue3 Dashboard 时间尺度卡片增强。
- Overview 子页完整时间尺度详情。
- 必要的 CSS 与契约校验。

## 验收标准

- 首页 24h / 3d / 7d 卡片不再只显示一句话，必须包含方向、强度、置信度、支撑驱动、压力驱动和观察摘要。
- Overview 子页必须展示完整的支撑驱动、压力驱动、主导侧、主导指标和观察规则。
- 指标展示优先使用中文可读标签，并保留 metric id 可追溯。
- 不修改 P1/P2/P3/P4.5 的数据契约。
- `npm run build` 与 `python scripts/validate_p5_dashboard_contract.py` 通过。
