# P5-C24 多源冲突证据展示与仲裁解释

## 状态

DONE

## 当前架构对齐（2026-05-22）

多源冲突页必须从 P1/P2 仲裁结果追到 P4.5 evidence。

展示字段：primary source、candidate/fallback source、source_resolution、fallback_used、fallback_reason、conflict_count、quality_score、downstream metric_score、metric_effective_score、downstream evidence_id。

页面需说明冲突是否影响 `final_view` 或仅作为 data boundary。

## 来源

P1-C22 / P1-C35 / P2-C21 已将多源冲突从“采集失败”治理为可审计的主源、fallback、候选源和冲突证据。P5 需要把这些字段展示出来，而不是把冲突重新渲染成错误。

```text
source_conflict: 5 个指标存在多源数值冲突
```

具体数量以本轮 P1/P2 HTML 与 P9 API 返回为准，不在 UI 写死。

这些冲突已经由 P1/P2 后端输出到 `conflicting_evidence.source_conflicts`，不是采集失败。下一步需要在 Evidence / Radar Detail / Data Quality 中可视化展示，让用户知道：

- 哪个源被选为主源
- 哪些源是 fallback / cross-check
- 差异有多大
- 差异属于口径差异、更新时间差异，还是疑似源数据异常

## 所属 Phase

P5 Vue3 Web / P2 Radar / P4 Evidence Pack / P9 API

## 目标

在 UI 中把多源冲突从“问题清单文字”升级为可审计证据块。

## 页面范围

### Evidence 证据页

增加 `多源冲突` 区块：

```yaml
source_conflict_card:
  metric_id: lightning_channel_count
  selected_source: mempool-lightning-network-stats
  conflicting_source: clarkmoody-dashboard
  conflict_type: definition_conflict
  severity: high
  relative_diff: 12.8%
  selected_reason: quality + priority + freshness
```

### Radar Detail 雷达详情页

在 Feature Calculation / Upstream Sources 中展示：

- selected source
- fallback source
- cross-check source
- conflict severity
- conflict type
- 是否影响当前雷达信号

### Data Quality 页面

多源冲突不应和采集失败混在一起，需要单独展示：

```text
数据可用，但多源口径不一致
```

## UI 要求

- `definition_conflict`：显示为口径差异，不用红色错误样式。
- `update_lag`：显示为更新时间差异，提示观察下一轮刷新。
- `value_conflict`：显示为黄色/红色风险，提示需要校准源优先级。
- 可点击进入 Source Detail。
- 可点击进入对应 Radar Detail。

## 后端/API 依赖

需要消费以下字段：

- `conflicting_evidence.source_conflicts`
- `features[].candidates`
- `features[].selected_reason`
- `features[].conflict`
- `freshness_policy`
- `collection_freshness_status`
- `business_recency_status`

若 P9 聚合 API 尚未提供这些字段，需要在 P9 对应任务中补充。

## DoD

- Evidence 页面能列出本轮全部多源冲突。
- Radar Detail 能解释每个冲突指标为什么选当前主源。
- Data Quality 页面能区分采集失败、数据滞后、多源冲突。
- P1/P2 本轮 source_conflict 能在 UI 中逐项追踪到。
- UI 不把 `definition_conflict` 渲染成采集失败。
- 页面 mock 测试覆盖：
  - low / medium / high 冲突
  - definition_conflict
  - update_lag
  - value_conflict
