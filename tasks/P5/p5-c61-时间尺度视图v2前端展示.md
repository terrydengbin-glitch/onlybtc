# P5-C61 / 时间尺度视图 v2 前端展示

## 状态

DONE

## 背景

当前时间尺度视图是摘要卡，不能清晰表达每个 horizon 的 acceptance、触发证据、反证证据、下一步确认和失效条件。新增 `btc_timescale_judge.v2.1` 后，前端需要升级成“时间尺度裁决卡”。

## 目标

只优化 Dashboard 的时间尺度视图区域，不修改 Radar 子页面、不修改拓扑雷达 div、不修改右侧抽屉整体结构。

## 前端消费优先级

1. `btc_timescale_judge.v2.1`
2. fallback: 旧 `horizon_views`
3. fallback: `btc_trend_cockpit.horizon`

## 每张卡显示

- horizon label：24h / 3d / 7d，4h 可作为 24h 内部 fast 子证据展示。
- direction + signal_stage
- effective_score + confidence_score
- acceptance state
- 支撑证据：trigger_eligible / confirmation
- 反证证据：rejected
- 背景证据：context_only / regime_only
- next confirmation
- next invalidation
- data quality flags

## 文案边界

- 不把 7d 写成短线入场信号。
- 不把 4h early warning 写成 confirmed。
- 不把 macro/fund_flow/context-only 单独写成确认方向。
- 必须保留“BTC 是否接受压力/支撑”的表达。

## DoD

1. Dashboard 时间尺度卡优先读取 `btc_timescale_judge`。
2. 缺新契约时旧 `horizon_views` 正常显示。
3. 24h/3d/7d 卡片展示 acceptance。
4. 支撑、压力、反证、质量降权视觉分离。
5. `npm run build` 通过。
6. 截图检查：卡片不溢出、不重叠、移动端可读。
