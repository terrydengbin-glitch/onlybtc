# P5-C05 事件倒计时与减产倒计时展示

## 状态

DONE

## 当前架构对齐（2026-05-22）

事件倒计时来自 P3 event window analyzer 和 P1 official macro calendar/fallback 数据。页面需要区分 pre-event watch、event day、post-event risk summary、fallback calendar source、Forecast/Actual/Previous 是否缺失。

FastAPI 读取：`GET /api/p3/events/latest`、`GET /api/macro/events/upcoming`。减产倒计时来自 P1 BTC block / halving metrics，并作为长期背景弱展示。

## 所属 Phase

P5

## 任务目标

展示宏观事件窗口、T-7 到 T+3 每日 watch、post-event 总结、FOMC/NFP/CPI 等事件倒计时，以及 BTC 减产倒计时。

## 背景依据

- [开发文档.md](../../开发文档.md)
- [task index.md](../../task%20index.md)

## 实施范围

- 展示事件类型、event_datetime、days_until、event_phase、window_action、publish_impact、daily_watch。
- T-7 到 T-3 范围内每次运行都显示“继续关注”状态与变化摘要。
- 标注事件来源是 parsed、embedded fallback 还是 provider fallback。
- 减产倒计时展示区分区块高度估算与日历日期估算。
- 遵守 evidence + data、历史窗口、数据质量、反证机制和预警边界。
- 不输出交易建议，不引入开仓、止损、仓位或杠杆逻辑。

## 输入

- P3-C14 事件窗口输出
- P1 官方宏观事件日历与 fallback 字段
- P1 BTC 区块高度/减产估算指标

## 输出

- 可运行或可复用的代码、配置、Schema、接口、组件或文档。
- 必要的测试、验证记录或运行说明。

## 验收标准

- 与《开发文档.md》的总体架构一致。
- 事件卡片必须能解释本次运行对事件的动作：monitor、watch、post_event_review 或 no_action。
- 关键状态、错误和数据质量可观测。
- 不绕过 P4.5 final_view、反证/确认规则、预警等级或数据质量约束。
