# P0-C07 数据源注册表与任务状态表设计

## 状态

DONE

## 所属 Phase

P0

## 任务目标

定义数据源注册和任务状态的领域语义、枚举、配置字段和接口草案，为 P8 SQLite Schema 提供输入。

注意：本任务不直接落地数据库表。数据库表、migration、Repository 由 P8 负责。

## 背景依据

- [开发文档.md](../../开发文档.md)
- [task index.md](../../task%20index.md)

## 实施范围

- 定义 source_id、source_group、method、update_frequency、fallback_source、validation_status 等字段语义。
- 定义任务状态枚举：queued、running、completed、partial_success、failed、blocked、cancelled。
- 定义 pipeline stage 枚举：fetching、cleaning、feature_calculation、radar_analysis、module_llm、fusion、multi_llm_debate、review、alert_policy、publish。
- 输出给 P8-C03 / P8-C05 使用的字段说明。
- 不创建 SQLite 表，不写 migration。

## 输入

- 上游 Phase 或前置任务产物。
- 开发文档中对应模块、雷达、总控、预警或 Dashboard 规范。

## 输出

- 数据源注册字段规范。
- 任务状态与阶段枚举规范。
- P8 Schema 输入说明。
- P0 空流程可使用的内存态/临时状态模型。

## 验收标准

- P8-C03 能根据本任务定义实现 sources / source_runs。
- P8-C05 能根据本任务定义实现 runs / run_stages。
- P1 数据源接入不需要重新定义 source 字段。
- P9 Run Logs API 不需要重新定义 stage/status 枚举。

## 依赖任务

P0-C05、P0-C06

## 备注

和 P8 的边界：P0-C07 定义语义，P8 负责持久化实现。
