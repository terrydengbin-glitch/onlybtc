# P3-C05 总控级反证条件检查器

## 状态

DONE

## 实现结果

已在 `backend/src/onlybtc/algorithms/p3.py` 实现 `check_global_invalidations()`。

支持四类总控反证：

- `bullish_state_invalidation`
- `bearish_state_invalidation`
- `data_quality_invalidation`
- `event_state_invalidation`

输出写入：

```yaml
tables:
  - invalidation_conditions
  - invalidation_events
payload:
  scope: global
  triggered_module_count
  near_module_count
  data_quality
  event_risk
```

## 行为

- 多个模块级反证触发时，总控进入 `triggered/near_trigger`。
- 数据质量 critical/warning 会触发总控质量反证。
- 事件风险过高会触发事件状态反证。

## 验收

- 支持 bullish / bearish / event / data quality 四类总控反证。
- 结果可驱动 P3-C06 预警分级和 P4 总控复核。
