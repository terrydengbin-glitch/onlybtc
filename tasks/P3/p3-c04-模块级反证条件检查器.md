# P3-C04 模块级反证条件检查器

## 状态

DONE

## 实现结果

已在 `backend/src/onlybtc/algorithms/p3.py` 实现 `check_module_invalidations()`。

每个 P2 雷达模块自动生成并检查两类模块反证：

- `{module_id}_data_quality_invalidation`
- `{module_id}_source_conflict_invalidation`

输出写入：

```yaml
tables:
  - invalidation_conditions
  - invalidation_events
payload:
  scope: module
  module_id
  trigger_distance
  evidence
```

## 状态规则

- `triggered`：模块 data_quality=low、存在 expired 核心指标或 source conflict。
- `near_trigger`：模块 data_quality=medium、低覆盖或缺少模块输出。
- `not_triggered`：模块质量和冲突状态正常。

## 验收

- 每个 P2 模块至少两条机器可执行反证条件。
- 反证事件带 `run_id/module_id/condition_id/status/evidence`。
- 可供 P5 Invalidation 页面展示。
