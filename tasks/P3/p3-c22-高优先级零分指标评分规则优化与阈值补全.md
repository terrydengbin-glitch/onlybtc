## 背景

P3-C21 已完成 Scoring Rulebook 审计，本轮抽样：

```text
指标 evidence 数 = 118
Radar module 数 = 14
zero 指标 = 64
zero ratio = 54.24%
初步优化候选 = 58
```

核心问题不是指标数量不足，而是：

1. `semantic.radar_rule` 承担了 67 个指标，通用规则过多。
2. 一批高价值主信号长期为 0，趋势敏感度不足。
3. 事件、风险、环境、上下文指标混在方向分里，语义不够清晰。
4. OI/Funding/Price/Volume、ETF flow、链上成本基础等指标需要组合解释，不能单项硬打分。

## 任务目标

第一批治理高优先级 0 分指标和容易误读的方向规则，让 P3 输出更适合 P4.5 研报和 Dashboard 展示。

本任务不是为了机械降低 zero ratio，而是让“应该对 BTC 趋势敏感的指标”真正产生可解释贡献，同时保留合理 0 分。

## 指标类型扩展

P3-C22 需要为指标补充或派生四类语义角色：

| 类型 | 作用 | 是否直接给方向 | 示例 |
| --- | --- | --- | --- |
| `direction_signal` | 直接判断 BTC 多空方向 | 是 | ETF flow、BTC 价格趋势、CVD、DXY、real yield |
| `risk_signal` | 表示风险锁定、置信度降级或发布谨慎 | 不一定 | CPI/FOMC/NFP 倒计时、VIX、IV、Fed speech risk |
| `regime_signal` | 判断市场环境、周期、风险偏好 | 弱方向 + 环境解释 | BTC dominance、ETH/BTC、Fear & Greed、MVRV |
| `context_signal` | 背景、审计、解释 | 否 | block height、supply、mempool pending fees |

规则：事件越近、风险越高，不应直接解释成 bullish/bearish，而应进入 `risk_lock`、`confidence_down` 或 `publish_tone=cautious`。

## P0 改造范围

### 1. fund_flow

目标：分离 ETF 绝对方向和边际动量。

新增或增强字段：

```text
flow_direction_score
flow_momentum_score
flow_state:
  bullish_inflow
  bearish_outflow
  bearish_but_improving
  bullish_but_fading
  mixed_flow
```

规则草案：

```text
ETF 净流入 > 0 -> flow_direction_score > 0
ETF 净流出 < 0 -> flow_direction_score < 0
3d/7d 流出收窄 -> flow_momentum_score > 0，但不能覆盖绝对净流出方向
3d/7d 流入缩小 -> flow_momentum_score < 0
```

验收重点：不能出现“ETF 仍净流出但被写成偏多”的逻辑冲突。

### 2. event_policy

目标：事件倒计时改为 risk overlay，不再硬给多空方向。

事件风险规则草案：

```text
0-1 天   -> event_risk_score = 0.9-1.0
1-3 天   -> event_risk_score = 0.7
3-7 天   -> event_risk_score = 0.4
7-14 天  -> event_risk_score = 0.2
14 天以上 -> event_risk_score = 0
```

输出：

```text
risk_lock
confidence_down
publish_tone = cautious / wait_confirm / no_chase
```

说明：宏观 surprise 在 Forecast/Actual/Previous 增强前，`macro_surprise_score=0` 继续保持中性。

### 3. derivatives_crowding

目标：OI / Funding / Price / Volume 组合解释，避免单项误判。

组合规则草案：

| 组合 | 解释 | 输出 |
| --- | --- | --- |
| 价格上涨 + OI 上升 + Funding 温和 | 新多进场，趋势确认 | 偏多 |
| 价格上涨 + OI 下降 | 空头回补，持续性一般 | 轻微偏多 |
| 价格下跌 + OI 上升 + Funding 为正 | 多头拥挤，踩踏风险 | 偏空 / risk 高 |
| 价格下跌 + OI 下降 | 杠杆出清 | 短线偏空但风险释放 |
| Funding 极高 + OI 高位 | 多头拥挤 | risk_score 上升，不必直接强空 |

输出：

```text
crowding_state
leverage_risk_score
derivatives_confirmation_score
```

### 4. onchain_valuation

目标：链上成本基础接 BTC price 相对位置。

组合规则草案：

| 条件 | 解释 | 分数 |
| --- | --- | --- |
| BTC > STH cost basis + 3% | 短持成本上方，趋势健康 | +0.15 ~ +0.25 |
| BTC 接近 STH cost basis ±2% | 关键成本区测试 | 0，但 risk_score 上升 |
| BTC < STH cost basis - 2% | 短持浮亏扩大 | -0.15 ~ -0.30 |
| BTC > realized price 且 MVRV 低位 | 长周期结构健康 | +0.10 ~ +0.20 |
| BTC 跌破 realized price | 结构性风险 | <= -0.30 |

说明：`cap_real_usd` 不建议直接给方向，优先使用 realized cap change 或作为 context。
