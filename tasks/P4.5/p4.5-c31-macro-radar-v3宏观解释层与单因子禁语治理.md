# P4.5-C31 / Macro Radar v3 宏观解释层与单因子禁语治理

## 状态

DONE

## 目标

让 P4.5 报告层正确解释 `macro_radar.v3`：它不是单因子宏观看多看空，而是 BTC 趋势的宏观确认、反证与冲击风险说明。

## 必须透传

```text
macro_trend_state
btc_implication
equity_beta
rates_pressure
dollar_pressure
volatility_stress
financial_stress
commodity_context
macro_impulse
btc_relative_confirmation
event_window
risk_score
confidence_adjustment
invalidation_conditions
```

## 禁语

报告层禁止输出：

```text
DXY 上涨，所以 BTC 看空。
VIX 高，所以 BTC 会跌。
Nasdaq 涨，所以 BTC 必涨。
黄金涨，所以 BTC 避险上涨。
油价涨，所以 BTC 看多/看空。
```

## 推荐表达

```text
美元和利率同时上行，说明外部金融条件转紧，BTC 当前反弹需要更强内部资金确认。
美股 beta 给出顺风，但 BTC 相对 Nasdaq 没有跟随，说明宏观顺风尚未被 BTC 吸收。
VIX 快速抬升不是方向信号，而是波动/去杠杆风险上升，当前 BTC 突破置信度需要下调。
宏观逆风下 BTC 仍保持正 residual，说明币圈内部买盘较强，不能仅凭 DXY/VIX 直接转空。
```

## DoD

- P4.5 final payload 包含 `macro_radar_explanation`。
- support/pressure/risk/context drivers 使用 v3 复合层语义。
- 单因子禁语不会出现在摘要、压力说明、模块解释中。
- 宏观风险与方向分离：risk drivers 不会伪装成 pressure direction。
- 旧报告 payload 缺少 v3 字段时可 graceful fallback。
