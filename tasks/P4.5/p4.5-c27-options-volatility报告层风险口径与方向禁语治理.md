# P4.5-C27 / Options Volatility 报告层风险口径与方向禁语治理

## 状态

DONE

## 背景

`options_volatility` 只提供风险与结构信号。如果报告层把 put-call、skew、max pain、gamma wall 写成直接看多/看空，会误导最终判断。

## 目标

报告层必须表达：

```text
Options Volatility 当前给出的不是方向信号，而是风险与结构信号。
当前状态为 {options_short_term_state}。
主要驱动来自 {risk_drivers}。
对交易层的影响是 {trade_permission_hint}。
本模块不会改变 final_direction，只调整 confidence、risk_mode 和 position_size。
```

## 禁止文案

```text
put_call_ratio 高，所以看空
options_skew 高，所以看空
max pain 在上方，所以看多
gamma wall 在下方，所以看空
IV 高，所以要跌
```

## 允许文案

```text
期权市场正在为未来波动付费。
下方保护需求升温，但不等于价格马上下跌。
短线靠近 gamma wall，突破信号需要打折。
大额到期临近，expiry 前后可能出现结构性波动释放。
IV 高于 RV，说明未来波动溢价上升，应该提高风险模式。
```

## DoD

- P4.5 Evidence / Final Writer / HTML Report 区分 direction drivers 与 options risk drivers。
- 报告不把 options 指标写成 bullish / bearish 直接依据。
- `options_volatility` 可调整 confidence/risk/trade_permission，但不改 final_direction。
- 报告层测试覆盖禁语。
