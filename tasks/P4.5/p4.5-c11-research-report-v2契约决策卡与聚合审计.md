# P4.5-C11 Research Report V2 契约、决策卡与聚合审计

## 状态

DONE

## 所属 Phase

P4.5 Radar Scored Analyst Writer

## 任务目标

在不破坏现有 `p45.final_article.v1`、C09 LLM 主研报和 C10 四分析师 LLM 分析的前提下，新增 `p45.research_report.v2` 契约，用确定性规则生成结构化决策卡、聚合审计、时间尺度拆分、反证条件和契约校验。

核心原则：

- v1 字段全部保留。
- v2 只新增结构化字段，不破坏旧 HTML 和旧 pipeline。
- LLM 负责解释，确定性代码负责评分、强度、置信度、反证条件和校验。
- 如果 P1/P2/P3 尚未提供某些增强字段，P4.5-C11 先以 warning 方式处理，不阻断 v2 产出。

## 背景

P4.5-C09/C10 已能生成真实 LLM 研究文章，但当前报告仍缺少成熟决策契约：

- 只有 `core_view=bearish`，没有强度。
- 没有明确 24h / 3d / 7d 周期拆分。
- 没有结构化反证条件。
- 没有解释为什么不是强空或强多。
- 没有聚合审计说明 14 个 module 和 118 条 evidence 如何汇总成最终方向。

## 实施范围

1. 决策卡 `decision_card`
   - `direction`
   - `direction_cn`
   - `strength`
   - `strength_cn`
   - `confidence`
   - `confidence_level`
   - `trade_permission`
   - `risk_mode`
   - `valid_horizon`
   - `conclusion_sentence`
   - `why_not_strong_bearish` / `why_not_strong_bullish`

2. 聚合审计 `aggregation_audit`
   - `final_score_raw`
   - `final_score_adjusted`
   - `direction`
   - `strength`
   - `confidence`
   - `disagreement_level`
   - `data_quality_level`
   - `score_components`
   - `dominant_drivers`
   - `counter_drivers`

3. 聚合公式
   - `metric_effective_score = metric_score * quality_score * freshness_weight * horizon_weight * duplicate_adjustment`
   - 当前若缺 `freshness_weight`，默认 `1.0` 并写入 warning。
   - 当前若缺 `duplicate_group_id`，先用内置 duplicate 规则识别已知重复指标。

4. 时间尺度拆分 `horizon_views`
   - `h24`
   - `d3`
   - `d7`
   - 每个周期包含：
     - `direction`
     - `strength`
     - `confidence`
     - `main_drivers`
     - `counter_drivers`
     - `interpretation`
     - `watch_rules`

5. 反证条件 `invalidation_rules`
   - 至少 3 条。
   - 覆盖 ETF 流出修复、1h 成交量修复、主动买盘持续强势等场景。
   - 每条包含：
     - `rule_id`
     - `title`
     - `horizon`
     - `metric_ids`
     - `current_state`
     - `trigger_condition`
     - `action_if_triggered`
     - `reason`

6. Contract validation
   - 如果 `zero_metric_ratio > 50%`，strength 不允许为 strong。
   - 如果存在明显 counter drivers，必须输出 `why_not_strong_*`。
   - 如果 unavailable metric count > 0，必须输出 data boundary。
   - 如果没有 `horizon_views` 或 `invalidation_rules`，validation failed。
   - 如果 P1/P2/P3 增强字段缺失，先输出 warning。
