# P4.5-C12 HTML Report v2 决策视图、周期视图、反证条件与发文层渲染

## 状态

DONE

## 所属 Phase

P4.5 Radar Scored Analyst Writer

## 任务目标

在 P4.5-C11 生成 `p45.research_report.v2` 后，升级 `reports/p45-research-report.html` 的展示结构。HTML v2 必须优先展示决策卡、时间尺度视图、反证条件、聚合审计和发文版本，同时保留当前所有 P4.5 内容，不覆盖、不删除 C09/C10 的 LLM 主研报和四分析师分析。

## 背景

当前 HTML 已经包含：

- run lineage
- deterministic final article
- deterministic analyst articles
- Radar Module 总分
- Evidence 附录
- Final / Analyst / Evidence Pack JSON
- C09 LLM 深度中文研报
- C10 四分析师 LLM 板块深度分析

但当前页面仍偏审计 dump，不适合快速判断：

- 顶部没有 direction / strength / confidence / trade_permission。
- 没有 24h / 3d / 7d 周期视图。
- 没有反证条件表。
- 没有聚合审计解释。
- 没有干净的发文版本。

## HTML v2 页面顺序

建议渲染顺序：

1. 顶部运行信息
   - 保留 run lineage。
   - 新增 direction / strength / confidence / trade_permission。

2. 决策卡
   - 方向。
   - 强度。
   - 置信度。
   - 交易许可。
   - 结论句。
   - 为什么不是强空/强多。

3. 时间尺度拆分
   - 24h
   - 3d
   - 7d
   - 每张卡展示：
     - direction
     - strength
     - confidence
     - main drivers
     - counter drivers
     - watch rules

4. 反证条件
   - 周期。
   - 指标。
   - 当前状态。
   - 触发条件。
   - 触发后动作。
   - 解释。

5. 聚合逻辑审计
   - final score。
   - penalties。
   - dominant drivers。
   - counter drivers。
   - duplicate / zero / unavailable / data quality warnings。

6. 发文版本
   - `publish_article.title`
   - `publish_article.body`
   - `safe_to_publish`
   - `forbidden_content_check`

7. 研究文章
   - 保留 C09 LLM 深度中文研报。
   - 保留 C10 四分析师 LLM 板块深度分析。

8. 旧 deterministic 基线
   - 保留 deterministic final article。
   - 保留 deterministic analyst articles。

9. Evidence 附录
   - 作为审计区保留。
   - 不让原始 evidence_id 污染发文版本。

10. JSON 附录
   - 保留 Final / Analyst / Evidence Pack JSON。
   - 新增 Research Report V2 JSON。

## 发文层契约

HTML 必须展示 `publish_article`：

```json
{
  "safe_to_publish": true,
  "publish_type": "binance_square_market_view",
  "style": "老马看盘",
  "title": "BTC 偏空震荡，别急着追空，先看资金流能不能修复",
  "body": "... $BTC",
  "cashtags": ["$BTC"],
  "forbidden_content_check": {
    "contains_raw_evidence_id": false,
    "contains_internal_run_id": false,
    "contains_python_dict": false,
    "contains_unexplained_score": false
  },
  "publish_score": 78,
  "reject_reason": null
}
```

强约束：

- `publish_article.body` 禁止出现 `p3-score`。
- 禁止出现 `run_id`、`collect_run_id`、`schema_version`。
- 禁止出现 Python dict 原样文本。
- 禁止堆原始 evidence_id。
- 必须包含 `$BTC`。
- 必须解释为什么偏空/偏多、为什么不是强信号、看什么反转。

## HTML Contract Validation

HTML 生成前必须检查：

- `has_decision_card`
- `has_horizon_views`
- `has_invalidation_rules`
- `has_publish_article`
- `publish_article_no_raw_evidence_id`
- `all_metrics_have_horizon_tags`
- `all_metrics_have_freshness`，缺失可 warning。
- `duplicate_groups_checked`

如果缺失核心字段：

- `decision_card`
- `horizon_views`
- `invalidation_rules`
- `publish_article`

则 HTML v2 不应静默成功，应在页面顶部显示 contract failure。

## 是否需要回到 P1/P2/P3 新增字段

P4.5-C12 主要是 HTML 渲染，不直接要求改 P1/P2/P3。

但 C12 会暴露字段缺口：

- 如果缺 `freshness_minutes`，HTML 显示 freshness warning。
- 如果缺 `horizon_tags`，HTML 显示 horizon warning。
- 如果缺 `duplicate_group_id`，HTML 显示 duplicate warning。

这些 warning 后续应拆为上游任务治理，但 C12 不阻塞。
