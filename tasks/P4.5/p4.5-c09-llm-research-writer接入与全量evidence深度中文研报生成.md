# P4.5-C09 LLM Research Writer 接入与全量 Evidence 深度中文研报生成

## 状态

DONE

## 所属 Phase

P4.5 Radar Scored Analyst Writer

## 任务目标

在现有 P4.5 deterministic 文章和 HTML 报告基础上，新增真正的 LLM Research Writer，让 LLM 读取 P4.5 Evidence Pack、四分析员文章、14 个 Radar module 总分和 118 条 scored metric evidence，生成一篇更接近专业 BTC 趋势研究报告的深度中文长文。

该报告必须追加到 `reports/p45-research-report.html` 尾部，不能覆盖、删除或替换当前 P4.5 已有内容：

- 最终综合研究文章
- 四位分析员文章
- Radar Module 总分
- Evidence 附录
- Final / Analyst / Evidence Pack JSON

## 背景

P4.5-C01 到 P4.5-C08 已经跑通新链路：

- P3 负责指标级正/零/负评分和 Radar module 总分。
- P4.5 Evidence Pack 已完整消费 14 个 Radar module 和 118 条 scored metric evidence。
- P4.5 HTML 已能展示文章、分析员输出、module 表和 evidence 附录。

但当前文章仍是 `runtime_mode=deterministic` 的模板化输出，`fallback_used=True`。它能作为审计摘要，但尚不能满足“LLM 读数据、感知趋势变化、给出专业推理文章”的目标。

## 实施范围

1. 新增 LLM Research Writer runtime
   - 建议新增 `backend/src/onlybtc/p45/llm_research_writer.py`。
   - 输入为当前 P4.5 full context：
     - final article payload
     - analyst articles payload
     - evidence pack payload
     - 14 个 module score / direction / explanation
     - 118 条 scored metric evidence
   - 输出独立 payload，例如 `p45_llm_research_article`。

2. Prompt 与输出要求
   - 输出中文。
   - 文章必须有清晰结构：
     - 核心结论
     - 趋势判断
     - 多空证据拆解
     - 关键冲突
     - 24h / 3d / 7d 观察路径
     - 数据边界和不可用指标说明
   - 必须引用 evidence_id 和关键数据值。
   - 必须结合 module score 和 metric score，不允许脱离 P3 分数自行裁判。
   - 必须说明“哪些信号正在变化”，而不是只罗列指标。

3. Evidence 使用策略
   - LLM 不要求逐字展开 118 条 evidence，但必须读取全量 evidence context。
   - 正文必须覆盖每个 Radar module 的关键信号。
   - 对未在正文展开的 evidence，仍保留在 HTML 附录。
   - 对 unavailable 指标必须说明为数据边界，不能当成负面趋势。

4. HTML 追加策略
   - 在 `reports/p45-research-report.html` 尾部新增 section，例如：
     - `LLM 深度中文研报`
     - `LLM Research Runtime`
     - `LLM Evidence Coverage`
   - 必须追加，不得覆盖或删除现有 P4.5 信息。
   - 原有 deterministic 文章继续保留，用作可审计基线。
   - 新增 LLM section 中的 evidence_id 也必须能跳转到 Evidence 附录。

5. Runtime 与降级治理
   - 支持 `runtime_mode=llm`。
   - 如果 LLM 失败，不覆盖 HTML，明确显示 LLM section 失败原因。
   - 不允许 silent fallback 伪装成 LLM 成功。
   - 需要记录 provider、model、status、error、created_at。

6. 验证与测试
   - 单测覆盖：
     - LLM payload schema
     - HTML 追加不覆盖旧 section
     - evidence_id 链接
     - LLM 失败时保留 deterministic 报告
   - 真实验收时需跑一次 P4.5 HTML，并确认尾部新增 LLM 深度研报 section。
