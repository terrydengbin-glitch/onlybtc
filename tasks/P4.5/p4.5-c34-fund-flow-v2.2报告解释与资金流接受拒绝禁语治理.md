# P4.5-C34 / Fund Flow v2.2 报告解释与资金流接受/拒绝禁语治理

## 状态

DONE

## Phase

P4.5 Research Report 与报告层

## 背景

`fund_flow.v2.2` 的语义从“资金流好坏”升级为“资金流是否被 BTC 接受”。P4.5 必须避免把 ETF 流入、稳定币增长或交易所余额下降写成单因子结论，也不能把 ETF 流出收窄写成 bullish。

## 目标

报告层解释必须明确区分：

```text
资金流 tailwind/headwind
BTC accepting/rejecting/resisting
warning vs confirmed
data-quality untrusted
```

## 文案规则

允许：

```text
ETF 流入正在形成边际需求，但是否构成趋势确认取决于 BTC response。
ETF 流出触发 fast warning，当前属于 trend_fragile，不等于趋势已确认转空。
稳定币供应扩张提供流动性背景，只能作为 mild tailwind。
交易所供给信号存在内部转账风险，本轮降权处理。
BTC 正在拒绝资金流顺风，说明内部卖压或外部压制更强。
BTC 正在抵抗资金流逆风，说明承接强度高于资金流模型预期。
```

禁止：

```text
ETF 流入所以 BTC 必涨。
稳定币供应增长所以看多 BTC。
交易所余额下降所以供给必然收缩。
ETF 净流出收窄所以 bullish。
单日交易所大额流出确认买盘。
```

## pressure_notes / warning_notes

当 `fund_flow_state` 为 warning 类状态时，增加 warning note：

```json
{
  "module": "fund_flow",
  "type": "fast_warning",
  "state": "etf_outflow_warning|etf_demand_fading|exchange_flow_untrusted",
  "message": "资金流出现早期预警，但尚未形成 confirmed 趋势结论。"
}
```

当 rejection 状态触发时，增加 rejection note：

```json
{
  "module": "fund_flow",
  "type": "flow_rejection",
  "state": "btc_rejecting_flow_tailwind|btc_resisting_flow_headwind",
  "message": "BTC 对资金流信号的响应与预期不一致，应优先解释为确认/反证关系。"
}
```
