# P4.5-C14 Horizon Driver 分类、聚合公式与反证规则结构化

## 状态

DONE

## 所属 Phase

P4.5 Radar Scored Analyst Writer

## 任务目标

修复 P4.5 v2 中聚合分数、周期 driver 分类和反证条件不可执行的问题，使方向评分、强度降级、周期驱动和反证规则具备一致、可解释、可审计、可机器消费的契约。

## 背景

当前 `aggregation_audit` 已存在，但存在两个核心问题：

1. 方向分数可能被 penalty 改符号。
   - 当前示例中 `net_score=-0.5087`，但 `final_score_adjusted=+0.0399`。
   - penalty 应该影响 strength / confidence，而不是把 bearish 分数推成 bullish/positive。

2. `horizon_views` 的 driver 分类有 bug。
   - 同一个 metric 可能同时出现在 `main_drivers` 和 `counter_drivers`。
   - 例如 `taker_buy_sell_ratio` 同时被写成主驱动和反向驱动。

此外，当前 `invalidation_rules` 多为自然语言字符串，对人可读但不能稳定机器执行。

## 实施范围

1. 聚合公式拆分
   - 将方向分数和置信度折扣拆开。
   - 建议字段：

```json
{
  "directional_score": -0.0333,
  "raw_net_score": -0.5087,
  "confidence_score": 0.4519,
  "strength_before_downgrade": "weak_bearish",
  "strength_after_disagreement": "neutral",
  "final_direction": "neutral",
  "downgrade_reasons": [
    "zero_metric_ratio_high",
    "support_pressure_disagreement_high",
    "horizon_disagreement"
  ]
}
```

2. 方向分数不允许被 penalty 改符号
   - disagreement penalty 只能影响：
     - `strength`
     - `confidence`
     - `trade_permission`
   - 不能把负向 directional score 改成正向 score。

3. Horizon driver 字段重构
   - 替换：
     - `main_drivers`
     - `counter_drivers`
   - 改为：

```json
{
  "support_drivers": [],
  "pressure_drivers": [],
  "dominant_side": "pressure",
  "dominant_drivers": []
}
```

   - `support_drivers` 表示支撑 BTC 的因子。
   - `pressure_drivers` 表示压制 BTC 的因子。
   - `dominant_drivers` 表示对当前 `final_view` 或该周期方向贡献最大的因子。

4. Driver 去重规则
   - 同一 `metric_id` 不允许同时出现在 `support_drivers` 和 `pressure_drivers`。
   - 如果同一指标跨模块重复，按 `duplicate_group_id` 合并或降权。

5. 反证规则结构化
   - 将自然语言字符串改为机器可执行 JSON 条件。
   - 每条规则至少包含：
     - `rule_id`
     - `applies_when`
     - `horizon`
     - `operator`
     - `conditions`
     - `action_if_triggered`
     - `reason`

6. 增加 bearish / bullish confirmation rules
   - 不只写“反证”，也要写趋势确认条件。
   - 示例：

```json
{
  "rule_id": "confirm_001_bearish_continuation",
  "title": "偏空延续确认",
  "applies_when": ["neutral", "weak_bearish"],
  "operator": "AND",
  "conditions": [
    {"metric_id": "etf_net_flow", "field": "metric_score", "op": "<", "value": 0},
    {"metric_id": "taker_buy_sell_ratio", "field": "metric_score", "op": "<", "value": 0},
    {"metric_id": "btc_1h_close", "field": "metric_score", "op": "<", "value": 0}
  ],
  "action_if_triggered": {"from": "neutral", "to": "weak_bearish"}
}
```

## 验收标准

- `directional_score` 不因 penalty 改符号。
- `strength` 可以因 disagreement / zero ratio 降级。
- `confidence` 可以因 disagreement / data quality 降低。
- `horizon_views` 使用 `support_drivers / pressure_drivers / dominant_side`。
- 同一 `metric_id` 不会同时出现在 support 和 pressure。
- `invalidation_rules` 不再只有自然语言 `trigger_condition`。
- 至少输出 3 条 invalidation rules。
- 至少输出 1 条 bearish confirmation rule 或 bullish confirmation rule。
- HTML 展示 support / pressure drivers。
- Contract validation 能检查 driver overlap。
