# P4.5-C17 P1/P2/P3/P4.5 含 LLM 一键全链条运行

## 状态

DONE

## 所属 Phase

P4.5 Radar Scored Analyst Writer

## 任务目标

新增一个统一的一键全量全链条运行入口，串联：

1. P1 真实数据采集与 P1-C22 HTML 验收报告
2. P2 Radar 生成与 Radar 质检 HTML
3. P3 指标评分、模块评分、算法审计 HTML
4. P4.5 deterministic Research Report v2
5. P4.5 LLM Research Writer 深度中文主研报
6. P4.5 四分析师 LLM 板块深度分析
7. 最终刷新 `reports/p45-research-report.html`

目标是让日常验证只需要一个命令完成全链条，不再手工分三步执行：

```powershell
python -m onlybtc.cli p45-full-audit --run-mode live --runtime-mode deterministic
python -m onlybtc.cli p45-llm-research-writer --final-run-id ... --runtime-mode llm --refresh-html
python -m onlybtc.cli p45-llm-analyst-writers --pack-id ... --runtime-mode llm --refresh-html
```

## 背景

当前 P4.5 已经进入稳定输出阶段：

- P1/P2/P3 能输出各自 HTML。
- P4.5 deterministic 报告已经包含 final_view、decision_card、aggregation_audit、horizon_views、invalidation_rules、publish_article 和 Evidence 附录。
- LLM Research Writer 可以读取 final payload 生成深度中文主研报。
- 四分析师 LLM 可以读取同一个 pack_id，为 14 个 Radar module 输出分工分析。

但实际运行仍需要人工复制 `final_run_id` 和 `pack_id`，容易出现：

- LLM 主研报和 deterministic final 不属于同一个 run。
- 四分析师 LLM 使用了旧 pack。
- HTML 已刷新但尾部 LLM 内容不是本轮。
- 验收时需要手工核对多个 run id。

因此需要一个统一 orchestrator，保证所有阶段使用同一轮 lineage。

## 实施范围

1. 新增 CLI 命令
   - 建议命名：`p45-full-audit-with-llm`
   - 或：`p45-run-all`
   - 参数建议：
     - `--run-mode live`
     - `--runtime-mode deterministic`
     - `--llm-runtime-mode llm`
     - `--skip-llm`
     - `--skip-analyst-llm`
     - `--skip-research-llm`
     - `--refresh-html`

2. 运行顺序
   - 先调用现有 P4.5 full audit 主链，生成：
     - `collect_run_id`
     - `p2_radar_run_id`
     - `p3_run_id`
     - `pack_id`
     - `article_run_id`
     - `final_run_id`
   - 再用本轮 `final_run_id` 调用 LLM Research Writer。
   - 再用本轮 `pack_id` 调用四分析师 LLM Writer。
   - 最后刷新 P4.5 HTML。

3. 输出汇总
   - 命令返回一个统一 JSON：

```json
{
  "status": "completed",
  "run_mode": "live",
  "runtime_mode": "deterministic",
  "llm_runtime_mode": "llm",
  "collect_run_id": "collect-xxx",
  "p2_radar_run_id": "radar-xxx",
  "p3_run_id": "p3-xxx",
  "pack_id": "p45pack-xxx",
  "article_run_id": "p45articles-xxx",
  "final_run_id": "p45final-xxx",
  "llm_research_run_id": "p45llm-xxx",
  "llm_analyst_run_id": "p45llmanalysts-xxx",
  "reports": {
    "p1": "reports/p1-c22-真实数据全链路验收报告.html",
    "p2": "reports/p2-radar-quality-report.html",
    "p3": "reports/p3-algorithm-audit-report.html",
    "p45": "reports/p45-research-report.html"
  },
  "contract_validation": {
    "status": "passed"
  },
  "llm_summary": {
    "research_status": "completed",
    "analyst_completed_count": 4,
    "analyst_failed_count": 0,
    "radar_modules_covered": 14
  }
}
```

4. Lineage 校验
   - LLM Research Writer 的 `final_run_id` 必须等于本轮 full audit 的 `final_run_id`。
   - 四分析师 LLM 的 `pack_id` 必须等于本轮 full audit 的 `pack_id`。
   - HTML 必须包含本轮：
     - `collect_run_id`
     - `p2_radar_run_id`
     - `p3_run_id`
     - `pack_id`
     - `final_run_id`
     - `llm_research_run_id`
     - `llm_analyst_run_id`

5. 降级策略
   - 如果 LLM 主研报失败，但 deterministic P1/P2/P3/P4.5 成功：
     - 命令可返回 `completed_with_llm_errors`。
     - HTML 保留 deterministic 内容，并展示 LLM 失败原因。
   - 如果四分析师 LLM 部分失败：
     - 允许保留已成功的分析师文章。
     - 汇总中标记失败分析师与 error。
   - deterministic 主链失败时，整个任务必须失败。

6. 日志与密钥安全
   - 输出汇总不得打印 `.env` key。
   - 如底层 HTTP log 包含外部 API URL，最终 CLI summary 不应回显敏感参数。

## DoD

- 新增一键 CLI 命令或脚本，可单命令跑完 P1/P2/P3/P4.5 deterministic + LLM 主研报 + 四分析师 LLM。
- 单命令输出包含完整 run lineage。
- P1/P2/P3/P4.5 四份 HTML 均生成。
- P4.5 HTML 尾部包含本轮 LLM 主研报与四分析师 LLM 板块。
- P4.5 final payload `contract_validation.status=passed`。
- 四分析师 LLM 覆盖 14 个 Radar module。
- 支持 `--skip-llm`，用于无 key 或节省成本时只跑 deterministic 主链。
- 单元测试通过。
- 至少跑一次真实 live 全链条验收。
