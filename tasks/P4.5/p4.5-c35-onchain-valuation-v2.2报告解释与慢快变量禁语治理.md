# P4.5-C35 / Onchain Valuation v2.2 报告解释与慢快变量禁语治理

## 状态

DONE

## Phase

P4.5 Research Report 与报告层

## 背景

`onchain_valuation.v2.2` 将输出 `regime_score`、`trend_delta_score`、`signal_stage`、`module_bias` 与 `invalidation_conditions`。报告层必须避免把 MVRV/NUPL、SOPR、STH 成本位等单因子写成确定多空结论。

## 目标

报告层解释必须明确区分：

```text
慢变量 regime
快变量 trend delta
early_warning vs fast_signal vs confirmed_signal
BTC accepting/rejecting/resisting
proxy exactness
```

## 允许文案

```text
MVRV/NUPL 显示链上估值背景偏热，但不能单独构成短线转空确认。
BTC 正在测试 STH 成本基础动态带，当前属于趋势关键位观察。
SOPR 回到 1 上方说明盈利恢复尝试，但是否确认需要价格响应与 residual 同向。
Realized Cap impulse 提供链上资本沉淀背景，仍需 BTC response 验证。
BTC 正在拒绝链上顺风，说明内部卖压或外部压制强于链上支撑。
本轮矿工/鲸鱼信号来自 proxy，只作为 context，不作为 confirmed signal。
```

## 禁止文案

```text
MVRV 高所以 BTC 必跌。
NUPL 低所以 BTC 必涨。
SOPR 大于 1 所以趋势确认转多。
BTC 站上 STH 成本位所以一定进入牛市。
Realized Cap 上升所以短线必涨。
鲸鱼/矿工 proxy 确认了链上抛压。
```
