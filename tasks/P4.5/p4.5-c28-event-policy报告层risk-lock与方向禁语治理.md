DONE

## 背景

`event_policy` v2.1 在报告层必须表达为事件风险门控，而不是宏观方向判断。报告应说明是否进入 caution、hard lock、post digest，以及 `trade_gate` 对新开仓、加仓、追突破和仓位倍数的限制。

## 目标

P4.5 Evidence / Final Writer / HTML Report 必须透传并解释：

```text
dominant_event_type

nearest_event_ts
nearest_event_hours
event_window_phase
event_risk_lock_level
risk_score
confidence_adjustment
trade_gate
risk_drivers
context_notes
```

## 禁止文案

```text
CPI 临近，所以看空 BTC
FOMC 临近，所以看跌
Fed speech 有风险，所以 bearish
blackout active，所以 BTC 方向转弱
```

## 允许文案

```text
CPI 临近，短线突破信心需要打折。
FOMC 进入 48h 风险窗口，建议降低加仓和追突破许可。
blackout active 表示政策沟通能见度下降，但不单独构成方向信号。
数据发布后需要等待首轮反应落地，再恢复正常交易许可。
```

## Summary 模板

```text
Event Policy 当前给出的不是方向信号，而是事件风险门控。
当前 dominant event 为 {dominant_event_type}，phase 为 {event_window_phase}。

```
