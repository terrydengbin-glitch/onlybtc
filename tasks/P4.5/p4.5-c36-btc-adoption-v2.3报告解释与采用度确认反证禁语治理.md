# P4.5-C36 / BTC Adoption v2.3 报告解释与采用度确认反证禁语治理

## 状态

DONE

## Phase

P4.5 研究报告与证据解释层

## 背景

`btc_adoption.v2.3` 不应被报告层写成“链上活跃上升所以看多”或“hashrate 上升所以看多”。报告层必须解释 fast/core/regime 的差异，以及 BTC 是否接受或拒绝链上采用度信号。

## 目标

P4.5 报告层消费 v2.3 payload，并形成专业、克制、趋势敏感的解释：

```text
链上采用度确认当前趋势
链上采用度削弱当前趋势
链上顺风被 BTC 拒绝
链上逆风被 BTC 抵抗
activity spike 可能是非经济噪音
fee pressure 是健康需求还是拥堵摩擦
hashrate/hashprice 只作为 regime 背景
```

## 禁语 / 防误导

禁止直接输出：

```text
active address 上升，所以 BTC 看多
transaction count 上升，所以 BTC 看多
hashrate 上升，所以短线 bullish
Lightning capacity 上升，所以趋势确认
fee 高，所以需求强
```

必须改写为：

```text
链上活跃是否伴随 adjusted transfer volume 和 BTC price response
费用压力是否由真实结算需求支持
hashrate/hashprice 仅说明网络安全和矿工压力背景
Lightning 仅作为中长期采用度 context
```

## DoD

- [ ] P4.5 报告可解释 v2.3 fast/core/regime 三层。
- [ ] 不把 raw activity、hashrate、Lightning level 单独写成方向结论。
- [ ] `activity_spike_untrusted` 必须解释为非经济活动风险。
- [ ] `btc_rejecting_adoption_tailwind` 必须解释为内部弱势或链上顺风未被接受。
- [ ] `btc_resisting_adoption_headwind` 必须解释为外部支撑或链上逆风被吸收。
- [ ] data_quality/proxy flags 被纳入置信度说明。
- [ ] P4.5 报告测试通过。
