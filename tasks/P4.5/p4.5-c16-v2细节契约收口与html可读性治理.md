# P4.5-C16 v2 细节契约收口与 HTML 可读性治理

## 状态

DONE

## 所属 Phase

P4.5 Radar Scored Analyst Writer

## 任务目标

在 P4.5-C13/C14/C15 已经完成 final_view 一致性、Horizon Driver 分类、Research Article v2 与 Publish Article 重写之后，继续收口 P4.5 v2 报告的细节契约，使报告从“准 DOD”进入稳定可用状态。

本任务不重构 P4.5 主架构，只修复以下细节：

- `why_not_strong` 不能为空，HTML 不再显示孤立 `-`。
- `horizon_views.interpretation` 必须严格区分 support / pressure driver，避免把压力指标写进偏多主因。
- `invalidation_rules.applies_when` 必须去重。
- `invalidation_rules.conditions` 与 `action_if_triggered` 在 HTML 中必须渲染为人类可读表达式，不能显示 Python dict 风格。
- `aggregation_audit` 必须解释 `raw_net_score -> directional_score` 的归一化逻辑。
- `contract_validation` 必须覆盖上述细节问题。

## 背景

最新 P4.5 报告已经解决上一版最大问题：

- `final_view` 成为唯一主裁判结论。
- `legacy_core_view` 已降级为历史兼容字段。
- `decision_card.direction`、`aggregation_audit.direction`、`research_article`、`publish_article` 已统一为 final view。
- `contract_validation` 已能捕捉 final view 冲突。

但仍存在若干契约细节：

- “为什么不是强单边”区域可能显示 `-`。
- 24h/3d/7d 周期文案可能把 pressure driver 混入 support 主因。
- `applies_when` 可能出现重复状态。
- HTML 反证条件仍可能显示 `{'metric_id': ...}` 这类 Python dict。
- 人类读者无法直接理解 `raw_net_score` 为什么归一化成 `directional_score`。

## 实施范围

1. 决策卡原因补全
   - `decision_card.why_not_strong` 必须至少 2 条。
   - HTML 决策卡优先读取 `why_not_strong`，再兼容旧字段。
   - contract 增加 `why_not_strong_reason_not_empty`。

2. Horizon 文案一致性
   - `direction=bullish` 时，主因只引用 `support_drivers`，压力另行说明。
   - `direction=bearish` 时，主因只引用 `pressure_drivers`，支撑另行说明。
   - `direction=neutral/mixed` 时，同时说明支撑和压力，不再写“主要由 X 驱动方向”。
   - contract 增加 `horizon_narrative_matches_driver_side`。

3. 反证规则规范化
   - `applies_when` 去重并保持稳定顺序。
   - `action_if_triggered.to` 必须使用允许状态枚举。
   - `counter_pressure_watch` 这类标签放入 `tag`，`to` 使用 `neutral_watch`。
   - contract 增加：
     - `applies_when_no_duplicates`
     - `invalidation_action_state_enum_valid`

4. HTML 可读化
   - `conditions` 渲染为 `metric.field op value`。
   - 多条件用 `AND` / `OR` 拼接。
   - `action_if_triggered` 渲染为 `from -> to (tag)`。
   - contract 增加 `html_no_python_dict_repr` 或等价检查。

5. 聚合归一化解释
   - `aggregation_audit.score_normalization` 新增：

```json
{
  "raw_net_score": -0.4681,
  "normalization_base": 15.8,
  "directional_score": -0.0296,
  "direction_threshold": {
    "weak_bearish": -0.12,
    "neutral_low": -0.12,
    "neutral_high": 0.12
  },
  "explanation": "raw_net_score 经过模块数量、权重上限、零分占比和分歧惩罚归一化后，落入 neutral 区间。"
}
```

## DoD

- P4.5 final payload 包含 `aggregation_audit.score_normalization`。
- `decision_card.why_not_strong` 非空，HTML 不显示孤立 `-`。
- 24h/3d/7d 的 interpretation 不再把 pressure driver 写成 bullish 主因，或把 support driver 写成 bearish 主因。
- `invalidation_rules.applies_when` 无重复。
- `action_if_triggered.to` 在允许状态枚举内，额外标签写入 `tag`。
- HTML 反证条件不再展示 Python dict 原样文本。
- `contract_validation.status=passed` 时必须包含并通过：
  - `why_not_strong_reason_not_empty`
  - `applies_when_no_duplicates`
  - `invalidation_action_state_enum_valid`
  - `horizon_narrative_matches_driver_side`
  - `html_no_python_dict_repr`
- 单元测试通过。
- 重新生成 `reports/p45-research-report.html` 并确认本任务字段出现在 HTML/JSON 中。
