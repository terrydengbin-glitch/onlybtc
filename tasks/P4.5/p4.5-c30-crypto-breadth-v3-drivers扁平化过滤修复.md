# P4.5-C30 / Crypto Breadth v3 drivers 扁平化过滤修复

## 状态
DONE

## 背景

Run once 审计发现：`crypto_breadth.v3` 的 `module_semantic_profile.support_drivers` 有值，但 P4.5 扁平化后的顶层 `support_drivers` 和 `crypto_breadth_explanation.support_drivers` 为空。

根因是 P4.5 flatten 阶段把 `_filter_btc_total_context_drivers()` 套用到了所有模块，导致 `crypto_breadth` 的 string driver 被误过滤。

## 目标

只对 `btc_total_state` 使用 halving / block height 专属过滤；其它模块必须保留自身 drivers 原始结构。

## 范围

- 修复 `backend/src/onlybtc/p45/final_writer.py` 中的 `support_drivers / pressure_drivers` 扁平化逻辑。
- 保持 `btc_total_state` 的 context/audit driver 禁入规则不变。
- 保持 `crypto_breadth.v3` 的 string drivers 可进入 final payload、dashboard API 和前端详情页。

## DoD

- `crypto_breadth` 顶层 `support_drivers` 不再为空。
- `crypto_breadth_explanation.support_drivers` 与 profile drivers 一致。
- `btc_total_state` 仍不会把 halving / block height 放入 direction drivers。
- P45 dashboard API 测试通过。
