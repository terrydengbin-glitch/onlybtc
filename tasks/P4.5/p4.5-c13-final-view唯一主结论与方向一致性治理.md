# P4.5-C13 Final View 唯一主结论与方向一致性治理

## 状态

DONE

## 所属 Phase

P4.5 Radar Scored Analyst Writer

## 任务目标

修复 P4.5 Research Report v2 中多套结论互相冲突的问题，引入唯一 canonical final view，使 Final JSON、HTML 顶部、决策卡、聚合审计、研究正文、发文层和 LLM 冲突提示使用同一套最终方向口径。

## 背景

当前报告已经具备 `decision_card`、`horizon_views`、`invalidation_rules`、`aggregation_audit`、`publish_article` 和 `contract_validation`，但存在严重方向冲突：

- `core_view = bearish`
- `decision_card.direction = neutral`
- `aggregation_audit.direction = neutral`
- deterministic research article 仍写“核心判断为偏空”
- LLM 深度研报仍写“核心观点：偏空”
- publish article 写“中性观察”

这说明当前 P4.5 有多套结论源，没有唯一主裁判结论。该问题会污染后续发文、交易节奏、审计判断和 Dashboard 展示。

## 实施范围

1. 新增唯一最终方向字段
   - 新增 `final_view`
   - 新增 `final_view_cn`
   - 新增 `decision_source`
   - 建议：

```json
{
  "final_view": "neutral",
  "final_view_cn": "中性观察",
  "decision_source": "aggregation_audit.directional_score_with_strength_downgrade"
}
```

2. 降级旧 core_view
   - 保留旧 `core_view` 兼容字段。
   - 新增 `legacy_core_view` 或 `p3_core_view`。
   - HTML 顶部不再把 `core_view` 当主结论展示。
   - 页面主结论只展示 `final_view`。

3. 统一结论字段
   - `decision_card.direction == final_view`
   - `aggregation_audit.direction == final_view`
   - `publish_article` 主结论必须与 `final_view` 一致。
   - deterministic `research_article` 必须按 `final_view` 生成。

4. LLM 观点冲突治理
   - 不强行改写 LLM 原文。
   - 如果 LLM research article 观点与 `final_view` 冲突，标记：

```json
{
  "llm_view_conflict": true,
  "llm_view": "bearish",
  "canonical_final_view": "neutral",
  "conflict_reason": "LLM article direction differs from final_view"
}
```

   - HTML 中提示：
     - “LLM 观点与量化聚合不一致，以下内容作为解释性参考，不作为最终决策。”

5. Contract validation
   - 新增 `view_consistency_check`
   - 新增硬错误 `FINAL_VIEW_CONFLICT`
   - 若存在冲突，`contract_validation.status = failed`

## 输出契约

Final JSON 增加：

```json
{
  "final_view": "neutral",
  "final_view_cn": "中性观察",
  "legacy_core_view": "bearish",
  "decision_source": "aggregation_audit.directional_score_with_strength_downgrade",
  "view_consistency_check": {
    "status": "passed",
    "conflicts": []
  }
}
```

## 验收标准

- HTML 顶部只显示 `final_view` 作为主结论。
- `core_view` 不再作为页面主结论。
- `final_view == decision_card.direction`
- `final_view == aggregation_audit.direction`
- `publish_article` 主结论与 `final_view` 一致。
- `research_article` 不再继续旧 bearish fallback 文案。
- 如果 LLM 文章与 `final_view` 冲突，HTML 明确显示冲突提示。
- `contract_validation` 能捕捉 `FINAL_VIEW_CONFLICT`。
- 若方向冲突存在，validation 必须 failed，不能 passed。
