# P4.5-C23 / Radar Module Detail API 透传 P3 复合语义字段

## 状态
DONE

## 背景

P3-C32 / P3-C33 已经在算法层补充了 derivatives_crowding 等模块的复合语义字段，例如：

- `module_effective_bias`
- `trend_direction`
- `crowding_state`
- `leverage_heat_state`
- `confirmation_state`
- `funding_state`
- `oi_state`
- `contribution_side`

但最新审计发现，`/api/p45/radar-modules/{module_id}` 和 Dashboard 最新 payload 的 Radar Module 投影仍会过滤部分字段，导致前端无法展示 P3 已经计算出的语义，只能退回到粗粒度 direction / score。

典型问题：

- `derivatives_crowding` 在 P3 已经能表达 “Funding 温和、OI flat、拥挤风险低、等待确认”。
- API detail 里 `btc_funding_rate` 没有透出 `funding_state / crowding_signal / direction_contribution / trend_confirmation`。
- API detail 里 `btc_open_interest` 没有透出 `oi_state / oi_confirmation / oi_trend_signal`。
- top contributors 没有稳定透出 `contribution_side`，前端容易把“正贡献”误读成“方向看多”。

## 目标

让 P4.5 Radar Module Detail API 与 Dashboard Radar Module 摘要完整透传 P3 复合语义字段，前端只负责展示，不重新推断。

## 修改范围

### P4.5 / API

1. `/api/p45/radar-modules/{module_id}` module payload 透传：
   - `module_effective_bias`
   - `trend_direction`
   - `trend_state`
   - `module_state`
   - `crowding_state`
   - `leverage_heat_state`
   - `confirmation_state`
   - `funding_state`
   - `oi_state`
   - `semantic_profile_version`

2. module metrics 透传：
   - `contribution_side`
   - `direction_contribution`
   - `funding_state`
   - `crowding_signal`
   - `trend_confirmation`
   - `oi_state`
   - `oi_confirmation`
   - `oi_trend_signal`
   - `flow_state`
   - `marginal_state`
   - `score_bucket_v2`
   - `zero_reason`

3. top contributors 透传：
   - `contribution_side`
   - `direction_contribution`
   - `funding_state`
   - `crowding_signal`
   - `trend_confirmation`
   - `oi_state`
   - `oi_confirmation`
   - `oi_trend_signal`

4. `/api/p45/dashboard/latest` 的 `radar_modules` 摘要保留必要复合状态字段：
   - `module_state`
   - `trend_state`
   - `module_effective_bias`
   - `fund_flow_state`
   - `fund_flow_absolute_direction`
   - `fund_flow_marginal_direction`
   - `crowding_state`
   - `leverage_heat_state`
   - `confirmation_state`

## 不改范围

- 不修改 P1/P2/P3 采集、评分、权重和组合规则。
- 不修改 P4.5 final_view 计算。
- 不修改 LLM 文章生成。
- 不在前端重新推断 Funding / OI / Fund Flow 语义。
- 不把 API projection 变成新的评分层。

- [x] `/api/p45/radar-modules/derivatives_crowding` 返回 module-level `module_effective_bias / crowding_state / leverage_heat_state / confirmation_state`。
- [x] `btc_funding_rate` metric 返回 `funding_state / crowding_signal / direction_contribution / trend_confirmation`。

- [x] API 缺少某个字段时保持兼容，返回 `null` 或省略，不导致 500。
- [x] 更新或新增 `backend/tests/test_p45_dashboard_api.py` 覆盖字段透传。
