# P4.5-C44 / BTC Acceptance Gate 时间尺度接受度

## 状态

DONE

## 背景

时间尺度视图不能只展示外部模块压力。每个 horizon 都必须判断 BTC 是否真正接受该周期的压力或支撑：外部偏空但 BTC 不跌，应降级为未确认或反证；外部偏多但 BTC 不涨，也不能升级为确认。

## 目标

新增 `btc_acceptance_gate`，供 `btc_timescale_judge.v2.1` 调用。

核心函数建议：

```python
compute_btc_acceptance(horizon, module_signals, price_features, flow_features)
```

## 输出字段

```json
{
  "state": "accepted|fragile|unconfirmed|rejected|blocked",
  "score": 0,
  "btc_response_score": 0,
  "residual_score": 0,
  "price_acceptance_score": 0,
  "flow_acceptance_score": 0
}
```

## 业务规则

1. bullish pressure 但 BTC 不涨，acceptance 只能 `unconfirmed|rejected`。
2. bearish pressure 但 BTC 不跌，acceptance 只能 `unconfirmed|rejected`。
3. external pressure 与 BTC residual 背离时，不允许 horizon confirmed。
4. micro / orderflow 可提升 4h/24h sensitivity，但不能替代 residual gate。
5. acceptance 缺失时，horizon 降级 `blocked` 或 `unconfirmed`，按数据质量决定。

## DoD

- 所有 horizon 均输出 acceptance。
- 24h / 3d confirmed 必须经过 acceptance gate。
- BTC response 缺失时不允许 confirmed。
- residual 与外部压力背离时输出 `why_not_stronger` 或 rejected evidence。
- 单测覆盖 bullish support rejected、bearish pressure absorbed、missing response downgraded。
