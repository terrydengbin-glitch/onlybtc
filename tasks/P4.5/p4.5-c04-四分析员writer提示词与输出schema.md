# P4.5-C04 四分析员 Writer 提示词与输出 Schema

## 状态

DONE

## 所属 Phase

P4.5 Radar Scored Analyst Writer

## 任务目标

实现 4 个轻量 Analyst Writer：宏观事件、流动性资金流、微观结构、链上结构。每个 Writer 只读取自己的 scored evidence，输出专业中文评论。

## 实施范围

- 定义 Analyst 输出 schema。
- 编写 4 个角色提示词。
- 约束必须引用 evidence_id。
- 要求解释正分、负分、零分和数据边界。
- 禁止输出交易指令。
- 支持 LLM 失败时 deterministic fallback，并显式标注。

## 验收标准

- 每个分析员输出一篇中文评论。
- 每篇评论至少引用本模块 5 个 evidence_id，若证据不足需说明。
- 每篇评论能回答“趋势是什么、为什么、看什么验证/反证”。

## 完成记录

已完成 4 个 Analyst Writer 的 deterministic runtime：

- 新增 `backend/src/onlybtc/p45/writer.py`。
- 支持读取 P4.5 Scored Evidence Pack。
- 输出并持久化：
  - `module_id=p45_analyst_articles`
  - `schema_version=p45.analyst_articles.v1`
- 4 个分析员均输出结构化中文评论：
  - `macro_event_analyst`
  - `liquidity_flow_analyst`
  - `microstructure_analyst`
  - `onchain_structure_analyst`
- 每篇评论包含：
  - `title`
  - `direction_view`
  - `score_summary`
  - `article`
  - 正/负/中性 evidence ids
  - `watch_items`
  - `data_boundary`
  - `prompt_frame`
  - `fallback_used`
- 当前先实现 deterministic fallback，后续可以在相同 schema 下接 LLM 扩写。
