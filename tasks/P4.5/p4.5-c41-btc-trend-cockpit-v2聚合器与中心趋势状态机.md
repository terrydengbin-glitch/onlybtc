## Trend Acceptance

```text
trend_acceptance_score =
  0.35 * price_response_score
+ 0.25 * residual_alignment_score
+ 0.25 * orderflow_confirmation_score
+ 0.15 * continuation_score

accepted:      score >= 65
unconfirmed:  45 <= score < 65
fragile:      25 <= score < 45
rejected:     score < 25
```

## 状态机硬约束

1. 中心 BTC 卡不直接读取 raw metric。
2. 单一 module 不能触发 `confirmed_bullish` 或 `confirmed_bearish`。
3. `risk_score` 高不能直接变 bearish。
4. `pressure_score` 高但 `trend_acceptance_score < 45` 时，最多 `bearish_watch`。
5. `support_score` 与 `pressure_score` 同时高时，必须 `conflict`。
6. BTC response/residual 缺失时不允许 confirmed signal。
7. data quality / contract blocking 时必须 `blocked`。
