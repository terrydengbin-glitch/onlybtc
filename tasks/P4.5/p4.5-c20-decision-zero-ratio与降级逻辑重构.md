# P4.5-C20 Decision Zero Ratio 与降级逻辑重构

## 状态

DONE

## Phase

P4.5 Research Report 与 P4 替代主线

## 背景

P4.5 当前会读取 P3 evidence 的 raw zero ratio，并可能用 `zero_metric_ratio_high` 作为全局降级原因。随着 P3 引入更多上下文指标、事件倒计时、组合指标和审计字段，raw zero 不再等价于规则缺口。

因此 P4.5 不应继续用 `raw_zero_metric_ratio` 直接压低最终 confidence。

## 目标

将 P4.5 降级逻辑从 raw zero 改为 decision zero：

```text
raw_zero_metric_ratio：只展示，不降级。
context_zero_ratio：不处罚，只提示上下文指标较多。
neutral_confirmed_ratio：不处罚，说明市场处于正常中性区间。
combo_required_ratio：只降低对应模块 confidence，不直接压全局。
rule_gap_zero_ratio：才真正处罚 P4.5 confidence。
```

`aggregation_audit.score_components` 中新增：

```json
{
  "raw_zero_metric_ratio": 0.0,
  "decision_zero_metric_ratio": 0.0,
  "rule_gap_zero_penalty": 0.0,
  "combo_required_module_penalty": 0.0
}
```

## 降级规则

```text
rule_gap_zero_ratio > 35%:
  confidence_down

decision_zero_metric_ratio > 40%:
  confidence_down

combo_required_ratio > 30%:
  module_confidence_down only

context_zero_ratio / neutral_confirmed_ratio:
  no confidence penalty
```

## HTML / Article 要求

- P4.5 HTML 显示 zero breakdown。
- 决策卡不再展示笼统 `zero_metric_ratio_high`。
- 研究文章需要区分：
  - “市场中性确认”
  - “上下文指标不参与方向”
  - “组合指标等待解读”
  - “规则缺口”
- Publish article 不出现内部字段名，但可自然表达“部分指标只是背景信息，不影响主判断”。

## DoD

- `raw_zero_metric_ratio` 不再直接作为 global downgrade reason。
- P4.5 final payload 包含 `zero_quality`。
- contract validation 新增：
  - `uses_decision_zero_ratio`
  - `raw_zero_not_used_as_penalty`
  - `zero_breakdown_present`
- P4.5 HTML 能看到 zero breakdown。
- P4.5 final_view / confidence 与旧契约兼容。
- P1/P2/P3/P4.5 全链条重跑通过。
