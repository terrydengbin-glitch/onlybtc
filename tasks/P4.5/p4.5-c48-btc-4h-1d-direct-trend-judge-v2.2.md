4h_direct_score =
  0.25 * price_structure_score

## 核心原则

1. Direct Evidence 决定方向。
2. Radar Context 只做确认、冲突、背景解释和最多 ±15 分 bias。
3. Event Overlay 默认只做 `trust_cap / confirmation_gate`，不直接做方向加减分。
4. Liquidity shock 默认降低 trust，不直接当作趋势。
5. Derivatives 必须使用 price × OI × taker_delta × funding 的条件语义。
6. BTC residual 必须输出业务语义，不只输出 z-score。
7. P4.5 confirmed 仍必须经过 acceptance/residual gate，不被 runtime 单点覆盖。

## Event Direction Rules

Event Window 可以影响 4h / 1d，但必须分阶段：

```text
pre_event:
  only trust cap
  no direction score change

post_event_unconfirmed:
  event_pressure_direction can appear
  max state = watch / building
  no confirmed

post_event_accepted:
  event pressure + BTC first reaction + 30m/2h followthrough + VWAP/range acceptance + CVD/taker confirmation
  can enter direction_score
```

反例：

```text
hot CPI + BTC first drop + 30m reclaim + CVD turns positive
=> shock_absorbed / btc_resilient
=> no confirmed bearish
```
