# P4.5-C10 四分析师 LLM 板块深度分析追加到 P4.5 报告

## 状态

DONE

## 所属 Phase

P4.5 Radar Scored Analyst Writer

## 任务目标

在 P4.5-C09 已完成的 LLM 深度中文研报之后，继续追加 4 位分析师对自己负责 Radar 板块的 LLM 深度分析。该内容必须追加到 `reports/p45-research-report.html` 尾部，放在 C09 主研报之后，不覆盖、不删除、不替换现有任何 P4.5 内容。

保留现有内容：

- deterministic final article
- 四分析员 deterministic article
- Radar Module 总分
- Evidence 附录
- Final / Analyst / Evidence Pack JSON
- P4.5-C09 LLM 深度中文研报

## 背景

P4.5-C09 已经让 LLM 读取全量 Evidence Pack，生成一篇综合中文研报。但当前 LLM 主研报是全局视角，虽然覆盖 14 个 Radar module，仍缺少 4 位分析师分别对自己负责板块的“专业分工分析”。

新的目标是让每位分析师只聚焦自己的 Radar 模块和 evidence：

- 宏观与事件分析师
- 流动性与资金流分析师
- 微观结构分析师
- 链上结构分析师

每位分析师输出一篇更细的中文板块分析，帮助读者理解各板块内部的多空证据、冲突、趋势敏感点和观察路径。

## 四分析师职责范围

1. 宏观与事件分析师
   - `macro_radar`
   - `treasury_credit`
   - `asia_risk`
   - `event_policy`

2. 流动性与资金流分析师
   - `dollar_liquidity`
   - `fund_flow`
   - `crypto_breadth`

3. 微观结构分析师
   - `kline_orderflow`
   - `derivatives_crowding`
   - `trade_structure_flow`
   - `options_volatility`

4. 链上结构分析师
   - `btc_total_state`
   - `btc_adoption`
   - `onchain_valuation`

## 实施范围

1. 新增四分析师 LLM Writer
   - 建议新增或扩展：
     - `backend/src/onlybtc/p45/llm_analyst_writer.py`
   - 输入来自 P4.5 Evidence Pack 中对应 analyst slice。
   - 每位分析师只消费自己负责的 Radar module 和 evidence。

2. 输出契约
   - 建议新增 SQLite 输出：
     - `module_id = p45_llm_analyst_articles`
     - `schema_version = p45.llm_analyst_articles.v1`
   - 每位分析师 payload 至少包含：
     - `analyst_id`
     - `title`
     - `runtime_mode`
     - `provider`
     - `model`
     - `status`
     - `article`
     - `radar_modules_covered`
     - `evidence_ids_used`
     - `metric_evidence_count_seen`
     - `error`

3. 文章要求
   - 中文输出。
   - 每位分析师必须引用 evidence_id、指标值、metric_score、module_score。
   - 必须解释自己板块的：
     - 核心结论
     - 正向证据
     - 负向证据
     - 零分/中性观察项
     - unavailable 数据边界
     - 24h / 3d / 7d 观察路径
   - 不得跨职责范围引用其他分析师的 evidence。
   - 不得给交易指令。

4. HTML 追加策略
   - 在 P4.5-C09 `LLM 深度中文研报` section 之后追加：
     - `四分析师 LLM 板块深度分析`
   - 必须追加，不允许覆盖现有内容。
   - 每位分析师一段独立 article card。
   - 每位分析师 evidence_id 必须能跳转到 Evidence 附录。
   - 若某位分析师 LLM 失败，该 card 显示 `status=failed` 和错误原因，其他分析师不受影响。

5. Runtime 与失败治理
   - 支持 `runtime_mode=llm`。
   - 支持 `runtime_mode=mock` 用于测试。
   - 不允许 silent fallback。
   - 失败结果必须落 SQLite，并在 HTML 中可见。

## 验收标准

- P4.5 HTML 原有内容全部保留。
- P4.5-C09 主 LLM 研报保留。
- HTML 尾部在 C09 后追加 4 位分析师 LLM 板块深度分析。
- 4 位分析师各自只覆盖自己的 Radar module。
- 4 位分析师文章均引用本板块 evidence_id。
- evidence_id 能从分析师文章跳转到 Evidence 附录。
- 单个分析师失败不影响其他分析师输出。
- `task index.md` 与任务卡保持同步。
