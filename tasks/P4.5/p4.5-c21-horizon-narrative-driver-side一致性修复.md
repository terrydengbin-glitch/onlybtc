# P4.5-C21 / Horizon Narrative Driver Side 一致性修复

## 状态
DONE

## 所属 Phase
P4.5 Research Report 与 P4 替代主线

## 背景
最新 run once 中 P1/P2/P3 链条正常，P4.5 deterministic final 已产出，LLM off 也按预期跳过。但 `/api/data-quality/latest` 暴露：

```text
contract_validation.status = failed
error = HORIZON_NARRATIVE_DRIVER_MISMATCH
item = d3 / bearish_not_pressure
```

根因是 `horizon_views` 的周期方向与主导驱动侧不一致：当周期方向为 bearish 时，`dominant_drivers` 仍可能因局部支撑贡献更大而选择 `support_drivers`，导致文案把压力方向写成支撑主因。

## 目标
修复 P4.5 周期视图文案生成逻辑，使 `horizon_views` 的 `direction`、`support_drivers`、`pressure_drivers`、`dominant_side`、`dominant_drivers`、`interpretation` 在 JSON、HTML、Dashboard API、Research Article 中保持一致。

## 实施结果
1. `direction` 为 bullish 时，`dominant_side` 优先固定为 `support`。
2. `direction` 为 bearish 时，`dominant_side` 优先固定为 `pressure`。
3. neutral / mixed 周期仍按支撑与压力贡献大小选择主导侧。
4. `dominant_drivers` 通过统一 helper 派生，避免方向与 driver side 打架。
5. `HORIZON_NARRATIVE_DRIVER_MISMATCH` 的错误信息补充 `expected_side`、`actual_side`、`conflicting_driver`。
6. 增加回归测试覆盖：d3 总分 bearish 但存在大额支撑驱动时，主导驱动仍必须来自 pressure side。
