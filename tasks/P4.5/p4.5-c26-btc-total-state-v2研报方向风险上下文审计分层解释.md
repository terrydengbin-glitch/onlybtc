# P4.5-C26 / BTC Total State v2 研报方向、风险、上下文、审计分层解释

## 状态
DONE

## 背景

P3-C41 会输出 `btc_total_state v2` 结构。P4.5 研报层需要消费该结构，并清晰区分：

```text
direction drivers
risk drivers
context notes
audit notes
```

避免把 Halving、Block Height、Funding、OI 写成错误的短线方向依据。

## 目标

P4.5 报告中将 BTC Total State 展示为：

```text
短线方向：price_state + perp_state 组合结论
合约确认：Funding / OI 对价格状态的确认或风险提示
周期背景：Halving context only
数据审计：Block height sync / missing / stale
```

## 禁止表达

报告不得出现以下误导：

```text
减半天数影响短线方向
区块高度影响 BTC 24h 判断
funding positive 所以 bullish
OI 高所以 bullish / bearish
```

## DoD

- P4.5 Evidence Pack 保留并透传 `btc_total_state v2` 字段。
- Final Writer / HTML Report 能渲染 `price_state`、`perp_state`、`cycle_context`、`audit_context`。
- Horizon drivers 不把 Halving / Block Height 放入 support/pressure drivers。
- Funding / OI 的文字必须体现 composite-only：需要结合 price_state。
- P4.5 tests 通过。

## 执行记录

- `backend/src/onlybtc/p45/evidence_pack.py` 保留 `affects_confidence / driver_eligible`，确保下游能识别 context/audit 指标。
- `backend/src/onlybtc/p45/final_writer.py` 透传 `btc_total_state v2` 分层字段，并新增 `btc_total_state_explanation`。
- Horizon `support_drivers / pressure_drivers` 过滤 `driver_eligible=false` 与 `affects_signal=false` 指标，避免 Halving / Block Height 进入方向驱动。
- `backend/src/onlybtc/p45/html_report.py` 新增 BTC Total State v2 分层解释区块，展示短线方向、合约确认、周期背景、数据审计与 composite-only 说明。
