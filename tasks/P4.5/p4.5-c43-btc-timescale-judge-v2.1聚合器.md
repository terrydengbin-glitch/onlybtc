# P4.5-C43 / BTC TimeScale Judge v2.1 聚合器

## 状态

DONE

## 背景

当前时间尺度视图主要是 `horizon_views` 摘要展示，结构偏薄：只有方向、阶段、分数、drivers/rejectors，不能独立回答每个周期内 BTC 是否接受压力、哪些证据可触发、哪些只是背景、哪些被反证或因质量降权。

本任务新增 `btc_timescale_judge.v2.1` 聚合器，把时间尺度视图升级为多周期趋势裁判层。

## 目标

新增 P4.5 层 `BTC TimeScale Judge v2.1`：

- 4h：fast sensing，负责敏感预警，不单独触发 headline confirmed。
- 24h：short trend confirmation，负责短线倾向是否成立。
- 3d：flow / macro / derivatives confirmation，负责资金与宏观延续确认。
- 7d：regime / background，负责中期背景，不覆盖短线。

## 输入

- `btc_trend_cockpit.v2`
- 14 个 radar module 的 module_signals / signal_stage / btc_implication / scores
- `invalidation_workbench.v2` 中可复用的 evidence 状态语义
- data quality / contract validation / run lineage

## 核心规则

1. 不允许简单平均 module score。
2. 每个 horizon 必须先区分 `trigger_eligible / confirmation / context_only / regime_only / rejected / quality_discounted`。
3. `context_only` 和 `regime_only` 不允许触发 confirmed signal。
4. 4h 只允许 early warning / fast signal / conflict / rejected / blocked，不允许单独让 headline confirmed。
5. 7d 只作为 regime cap / background，不允许覆盖 24h 快速变化。
6. BTC response / residual 缺失时，horizon confirmed 自动降级。
7. 输出必须兼容旧 `horizon_views` fallback。

## DoD

- run once 后 `final_payload.btc_timescale_judge.schema_version = p45.btc_timescale_judge.v2.1`。
- 输出 4h / 24h / 3d / 7d 四个 horizon。
- 每个 horizon 有 raw/effective/confidence/sensitivity/stability score。
- 每个 horizon 有 evidence 分类与 summary。
- 单一慢变量或 context-only 模块不能触发 confirmed。
- 后端单测覆盖 4h 不单独 confirmed、7d 不覆盖短线、缺 BTC response 降级。
