# P4.5-C45 / Cross-Horizon Arbiter 多周期仲裁器

## 状态

DONE

## 背景

4h、24h、3d、7d 的语义不同，不能直接平均。需要独立仲裁层判断多周期是否共振、背离、冲突或被数据质量阻断。

## 目标

新增 `cross_horizon_arbiter`，接收 `btc_timescale_judge.horizons`，输出 headline bridge。

## 输出契约

```json
{
  "dominant_horizon": "4h|24h|3d|7d",
  "alignment": "aligned|divergent|conflict|blocked",
  "headline_direction": "bullish|bearish|neutral",
  "headline_stage": "watch|confirmed|rejected|conflict|blocked",
  "why_not_stronger": "",
  "why_not_reversed": ""
}
```

## 仲裁规则

1. 4h strong + 24h neutral => watch，不允许 headline confirmed。
2. 24h + 3d 同向且 acceptance accepted => headline confirmed。
3. 7d 只影响 regime cap，不直接覆盖 24h。
4. 7d supportive + 24h bearish => 中期顺风下短线回撤，不写 confirmed bearish。
5. 7d pressure + 24h bullish => 逆风背景下短线反弹，不写 confirmed bullish，除非 3d 同向确认。
6. 任何核心数据质量 blocking，headline_stage = blocked。

## DoD

- 输出 cross_horizon 字段。
- 4h 无法单独触发 headline confirmed。
- 24h/3d 同向 + accepted 可升级 confirmed。
- 7d 不覆盖短线。
- conflict 时必须给出 `why_not_stronger`。
- replay/history 中能复现当时 cross_horizon 结论。
