# P6-C04 预警历史与预警质量评分

## 状态

DONE

## 所属 Phase

P6 文章生成、历史记录与回测评分

## 任务目标

在 P3 algorithm alerts 的基础上建立 P6 预警历史和质量评分视图。该视图只评估预警的可解释性、证据完整性和生命周期可观测性，不改变预警触发逻辑、不升级/降级交易判断。

## 背景依据

- [开发文档.md](../../开发文档.md)
- [task index.md](../../task%20index.md)
- P3 algorithm alerts / alert_events
- P6-C03 总控判断历史与 Evidence Pack 回放
- P9-C05 Alerts 与 Invalidation 聚合 API

## 实施范围

- 新增 P6 alert history API：
  - `GET /api/p6/alerts/history`
- 新增 P6 alert quality API：
  - `GET /api/p6/alerts/quality`
- response contract：
  - `schema_version=p6.alert_history.v1`
  - `schema_version=p6.alert_quality.v1`
  - history item 包含 `alert_id/run_id/level/state/title/summary/evidence_count/event_count/updated_at`
  - quality item 包含 `quality_score/status/checks/issue_count`
- 质量评分只用于 P6 回放和后续评分任务，不写回 P3 alert 表。
- 遵守 evidence + data、历史窗口、数据质量、反证机制和预警边界。
- 不输出交易建议，不引入开仓、止损、仓位或杠杆逻辑。

## 输入

- SQLite `algorithm_alerts`
- SQLite `alert_events`
- P3/P9 alert contract

## 输出

- P6 alert history query service
- P6 alert quality scoring service
- FastAPI read endpoints
- focused tests

## 验收标准

- [x] `GET /api/p6/alerts/history` 返回预警历史列表和只读历史模式。
- [x] `GET /api/p6/alerts/quality` 返回预警质量评分摘要。
- [x] Quality checks 覆盖：level、state、title、summary、evidence_count、lifecycle event。
- [x] 空数据返回 `status=empty`，不报 500。
- [x] 不修改 P3 alert rows，不改变预警级别或状态。
