# P6-C01 自动文章生成流程

## 状态

DONE

## 所属 Phase

P6 文章生成、历史记录与回测评分

## 任务目标

基于 P4.5 Report v2 / P9 页面契约，建立自动文章生成的 deterministic draft snapshot 流程。该流程只生成“解释层文章草稿”，不发布、不交易、不改变 P4.5 final 判断，供 P6-C02 手动发布策略、P6-C03 历史回放、P6-C07 DoD 继续复用。

## 背景依据

- [开发文档.md](../../开发文档.md)
- [task index.md](../../task%20index.md)
- P9-C03 Article/Evidence 聚合 API
- P9-C12 FastAPI 页面契约验收
- P9-C13 API Mock 与 DoD 验收

## 实施范围

- 新增 P6 自动文章 snapshot builder。
- 输入 frozen P4.5 final payload，不读取当前 realtime 状态覆盖历史快照。
- 输出 `p6.auto_article.v1`，包含：
  - `article_snapshot_id`
  - `final_run_id`
  - `pack_id`
  - `article_run_id`
  - `source_schema_version`
  - `draft_status`
  - `title`
  - `summary`
  - `body`
  - `evidence_citations`
  - `risk_disclosures`
  - `quality_gate`
  - `publish_boundary`
  - `run_lineage`
- 持久化到 `module_json_outputs`，`module_id=p6_auto_article_snapshot`。
- P4.5 full-chain job 在 deterministic final 完成后自动生成 P6 draft snapshot。
- 不输出交易建议，不引入开仓、止损、仓位或杠杆逻辑。

## 输入

- P4.5 final payload：`p45.research_report.v2`
- P9 article/dashboard/history API contract
- `metric_evidence`
- `contract_validation`
- `data_quality`

## 输出

- `onlybtc.p6.article_pipeline`
- `p6.auto_article.v1` snapshot
- focused tests
- P4.5 full-chain job result 中的 `p6_article_snapshot_id`

## 验收标准

- [x] 可从 P4.5 final payload 自动生成 P6 article snapshot。
- [x] 同一 `final_run_id` 重复生成保持幂等，返回同一个 `article_snapshot_id`。
- [x] snapshot 引用 evidence_id，且 citation 不超出 final payload 的 `metric_evidence`。
- [x] `publish_boundary` 明确 `auto_publish_allowed=false`。
- [x] `quality_gate` 可观测 contract/data/evidence/body 状态。
- [x] P4.5 full-chain job 自动生成 P6 draft snapshot 并记录 lineage。

## 执行记录（2026-06-23）

- 新增 `onlybtc.p6.article_pipeline`。
- 新增 P6 schema 常量：
  - `P6_AUTO_ARTICLE_MODULE_ID = p6_auto_article_snapshot`
  - `P6_AUTO_ARTICLE_SCHEMA_VERSION = p6.auto_article.v1`
- 新增 `build_auto_article_snapshot(final_payload)`：
  - 从 frozen P4.5 final payload 构建 deterministic draft。
  - 输出 evidence citations、risk disclosures、quality gate、publish boundary。
  - 明确 `auto_publish_allowed=false`，发布动作留给 P6-C02。
- 新增 `generate_auto_article_snapshot(final_run_id=None, db=...)`：
  - 从 `module_json_outputs` 读取 P4.5 final payload。
  - 写入 `module_json_outputs`，`module_id=p6_auto_article_snapshot`。
  - 以 `p6article-{final_run_id}` 保持幂等。
- 新增 `latest_auto_article_snapshot(db=...)`。
- P4.5 full-chain job 在 deterministic final 完成后调用 P6 builder，并在 job result 中记录 `p6_article_snapshot_id`。
