# P6-C05 24h/72h/7D 结果跟踪

## 状态

DONE

## 所属 Phase

P6 文章生成、历史记录与回测评分

## 任务目标

基于 P6 article snapshot 建立 24h / 72h / 7D 后续结果跟踪。跟踪只描述文章快照后的 BTC 价格变化和 final view 的观察性对齐状态，不输出交易建议，不生成开仓/止损/仓位或杠杆逻辑。

## 背景依据

- [开发文档.md](../../开发文档.md)
- [task index.md](../../task%20index.md)
- P6-C03 总控判断历史与 Evidence Pack 回放
- P6-C04 预警历史与预警质量评分
- P8-C04 时序指标 metric_values 与历史窗口索引

## 实施范围

- 新增 P6 outcome tracking API：
  - `GET /api/p6/outcomes/track`
- 支持参数：
  - `article_snapshot_id`
  - `limit`
  - `run_mode`
- response contract：
  - `schema_version=p6.outcome_tracking.v1`
  - `items[].article_snapshot_id`
  - `items[].final_run_id`
  - `items[].final_view`
  - `items[].anchor_price`
  - `items[].horizons.24h/72h/7d`
- 每个 horizon 输出：
  - `status=observed|pending|missing_anchor|missing_target`
  - `target_at`
  - `target_price`
  - `return_pct`
  - `directional_alignment`
- 结果跟踪为只读，不写回 P4.5 final，不修改 P6 article snapshot。
- 遵守 evidence + data、历史窗口、数据质量、反证机制和预警边界。
- 不输出交易建议，不引入开仓、止损、仓位或杠杆逻辑。

## 输入

- `p6.auto_article.v1`
- `p45.research_report.v2`
- SQLite `metric_values` 中 `btc_price` / `btc_close`

## 输出

- P6 outcome tracking query service
- FastAPI read endpoint
- focused tests

## 验收标准

- [x] `GET /api/p6/outcomes/track` 返回 P6 article snapshots 的 24h/72h/7D 结果跟踪。
- [x] 支持按 `article_snapshot_id` 过滤。
- [x] 价格缺失或 horizon 未到期时返回可观测状态，不报 500。
- [x] 返回只读边界：`read_only=true`、`mutates_final_view=false`、`trading_advice=false`。
- [x] 不修改 P4.5 final、不修改 P6 article snapshot、不写交易建议。
