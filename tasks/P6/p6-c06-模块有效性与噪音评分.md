# P6-C06 模块有效性与噪音评分

## 状态

DONE

## 所属 Phase

P6 文章生成、历史记录与回测评分

## 任务目标

基于 P6-C05 outcome tracking 和 P4.5 `radar_module_scores` 建立模块有效性与噪音评分视图。该视图只用于历史回放归因和后续 P6-C07 DoD 汇总，不修改雷达模块权重、不改变 P4.5 final 判断。

## 实施范围

- 新增 P6 module effectiveness API：
  - `GET /api/p6/modules/effectiveness`
- 支持参数：
  - `article_snapshot_id`
  - `limit`
  - `run_mode`
- response contract：
  - `schema_version=p6.module_effectiveness.v1`
  - `modules[].module_id`
  - `modules[].observed_count`
  - `modules[].aligned_count`
  - `modules[].miss_count`
  - `modules[].effectiveness_score`
  - `modules[].noise_score`
  - `modules[].status`
- 评分只使用已 observed 的 outcome horizons；pending/missing 不参与有效性分母，但计入可观测 gap。
- 不反写 P3/P4.5/P6 数据，不更新任何模块权重。
- 遵守 evidence + data、历史窗口、数据质量、反证机制和预警边界。
- 不输出交易建议，不引入开仓、止损、仓位或杠杆逻辑。

## 验收标准

- [x] `GET /api/p6/modules/effectiveness` 返回模块有效性和噪音评分。

- [x] 只使用 observed horizons 评分；pending/missing 作为 gap 可观测。

## 实施记录

2026-06-23：

- 新增 `onlybtc.p6.module_effectiveness`：
  - `P6_MODULE_EFFECTIVENESS_SCHEMA_VERSION = p6.module_effectiveness.v1`
  - `module_effectiveness(article_snapshot_id=None, limit=50, run_mode=live)`
- 新增 FastAPI endpoint：
  - `GET /api/p6/modules/effectiveness`
- 评分输入：
  - P6-C05 `outcome_tracking`
  - P4.5 final payload `radar_module_scores`
- 评分逻辑：
  - 只使用 `status=observed` 的 24h/72h/7d horizons。
  - 根据模块方向/分数与 observed return 方向对齐情况计算。
  - `effectiveness_score = aligned_count / scored_count`
  - `noise_score = 1 - effectiveness_score`
  - pending/missing horizon 不进入分母，只计入 gap。
- 只读边界：
  - `read_only=true`
  - `mutates_module_weights=false`
  - `trading_advice=false`
