## 任务定位

本任务只补“无需付费、无需购买 provider”的数据源，并把指标分成 `exact`、`proxy`、`event` 三层，避免后续 LLM 把代理数据误当成精确事实。

P1-C14 继续负责 Glassnode 公开数据与登录态勘探；P1-C15 负责把 P2 当前缺口中可以免费关闭的部分落地，并为不能免费精确解决的指标建立清晰代理口径。

## 免费可关闭缺口

### 1. Lightning Capacity

不要继续用 Glassnode Lightning 匿名页面作为主源。Glassnode 匿名返回 403，生产价值低。

```yaml
lightning_capacity:
  primary:
    source: mempool_space
    method: rest_or_mempool_js_equivalent
    fields:
      - total_capacity
      - channel_count
      - node_count
  fallback:
    source: mempool_space_lightning_page
    method: playwright_text
```

实施要求：

- 新增 `mempool-lightning-network-stats` source。
- 输出 `lightning_capacity_btc`、`lightning_channel_count`、`lightning_node_count`。
- 接入 BTC 采用率雷达。
- 若 API TLS/网络失败，降级到 Playwright 页面文本，不阻塞主链路。

### 2. Realized Price

不再依赖 Glassnode 匿名 `price_realized_usd`。主源改为 Coin Metrics Community 免费数据。

```yaml
realized_price:
  primary:
    source: coinmetrics_community
    method: community_api_or_csv
    csv: https://raw.githubusercontent.com/coinmetrics/data/master/csv/btc.csv
    required_fields:
      - CapMrktCurUSD
      - CapMVRVCur
      - SplyCur
    formula:
      cap_real_usd: CapMrktCurUSD / CapMVRVCur
      realized_price: cap_real_usd / SplyCur
  fallback:
    source: glassnode
    endpoint: /v1/metrics/market/price_realized_usd
    requirement: api_key
```

实施要求：

- 新增 `coinmetrics-community-btc-csv` source。
- 读取 `CapMrktCurUSD`、`CapMVRVCur` 与 `SplyCur`，计算 `cap_real_usd = CapMrktCurUSD / CapMVRVCur`，再计算 `realized_price = cap_real_usd / SplyCur`。
- 保存原始字段：`cap_real_usd`、`supply_current`、`realized_price`。
- 接入链上估值与筹码雷达。

### 3. Liquidation Long / Short

```yaml
liquidation_long_short:
```

实施要求：

- 新增 Binance USD-M `forceOrder` collector。
- 新增 Bybit V5 `allLiquidation.BTCUSDT` collector。
- 标准化输出：

  - `liquidation_snapshot_window_ms`
- 明确质量说明：交易所级实时强平，不等于全市场精确清算总量。

实施要求：

## 免费代理指标

### 1. Exchange Netflow Proxy

当前 Glassnode 交易所余额日变化不能继续叫精确 `exchange_netflow`。

```yaml
exchange_balance_delta_1d_proxy:
  current_source: glassnode_public_balance
  formula: exchange_balance_today - exchange_balance_yesterday
  confidence: medium
  note: 余额变化代理，不等于 exchange_inflow - exchange_outflow

exchange_netflow_exact:
  provider_required:
    - cryptoquant
    - glassnode
```

实施要求：

- 将当前代理指标在文档、metric definition、UI data quality 中标记为 `proxy`。
- 保留旧消费兼容层，但不允许新模块把 proxy 当 exact。

### 2. Gamma Wall Proxy

当前 Deribit OI 近似应改名。

```yaml
gamma_wall_proxy_distance:
  source: deribit
  method:
    - get_book_summary_by_currency
    - public/ticker greeks when available
  output:
    - oi_max_strike
    - gamma_oi_concentration_strike
    - nearest_high_oi_strike_distance
  label: proxy
```

实施要求：

- 将现有 `gamma_wall_distance` 的近似结果改为 `gamma_wall_proxy_distance`。
- `gamma_wall_distance_exact` 标记为 provider-required。
- 期权雷达解释中必须显示“proxy，不是专业 dealer gamma wall”。

### 3. Miner Flow Proxy

免费路径可以做弱代理，但不要宣称全市场精确矿工流。

```yaml
miner_exchange_flow_proxy:
  free_proxy:
    source:
      - glassnode_public_hashrate_and_miner_context
      - public_miner_wallet_watchlist_if_available
  exact_requires:
    - cryptoquant
    - glassnode
```

实施要求：

- 若无稳定免费矿工流 API，只登记 proxy 结构和空值状态。
- 不用不稳定截图/图表 OCR 直接喂给 P2 强判断。

### 4. Whale Flow Proxy

`whale_flow` 必须拆成更明确的代理项。

```yaml
whale_exchange_flow_proxy:
  components:
    - large_tx_alert
    - exchange_inflow_top10_proxy
    - exchange_whale_ratio_proxy
  confidence: low_to_medium
  exact_requires:
    - cryptoquant
    - entity_label_provider
```

实施要求：

- 新 metric 不叫泛化 `whale_flow`。
- 若无免费稳定源，先只保留 schema、质量说明和 provider-required 标记。

### 5. Stablecoin Exchange Inflow Proxy

免费代理只能代表稳定币供给或链上蓄水池，不能替代交易所流入。

```yaml
stablecoin_buying_power_proxy:
  source:
    - defillama_stablecoins
  fields:
    - stablecoin_total_supply
    - stablecoin_supply_7d_change
  note: 不是 stablecoin_exchange_inflow

stablecoin_exchange_inflow_exact:
  provider_required:
    - cryptoquant
    - glassnode
```

实施要求：

- 可新增 DeFiLlama stablecoins 免费源，输出 `stablecoin_total_supply` 与 `stablecoin_supply_7d_change`。
- 不命名为 `stablecoin_exchange_inflow`。
- P2 交易结构雷达只把它作为辅助流动性证据。

## 备注

本任务的核心不是“更多抓数据”，而是把数据置信层级理顺：exact 可以参与强判断，proxy 只能辅助，event 负责提醒。这样 P2/P3/P4 才不会因为一个弱代理指标而产生过度确定的结论。
