# P1-C08 历史数据存储与窗口计算基础字段

## 状态

DONE

## 所属 Phase

P1

## 任务目标

基于 P8-C04 的 `metric_values` 时序表，定义历史窗口计算所需字段、查询窗口和特征输入规范。

注意：历史数据物理存储由 P8 负责，本任务不再单独设计数据库表。

## 背景依据

- [开发文档.md](../../开发文档.md)
- [task index.md](../../task%20index.md)

## 实施范围

- 定义历史窗口：24h、7d、30d、90d、1y。
- 定义窗口计算输入字段：metric_id、source_id、observed_at、value_num、timeframe、quality_score、is_fallback。
- 定义输出字段：change_24h、change_7d、rolling_mean、rolling_std、z_score、percentile。
- 明确哪些指标必须做趋势判断。
- 依赖 P8-C04 的 metric_values，不另建存储。

## 输入

- 上游 Phase 或前置任务产物。
- 开发文档中对应模块、雷达、总控、预警或 Dashboard 规范。

## 输出

- 历史窗口字段规范。
- 窗口计算查询需求。
- P3 算法检测所需特征字段清单。
- P2 雷达模块读取历史窗口的接口约定。

## 验收标准

- P2 雷达模块能按统一窗口读取历史数据。
- P3 z-score / 分位数 / 背离检测有明确输入字段。
- 趋势敏感指标不会只读取当前值。
- 所有窗口计算能携带 data_quality / fallback 信息。

## 依赖任务

P8-C04、P1-C07

## 备注

和 P8 的边界：P8 负责存储和索引，P1-C08 负责定义窗口计算语义和特征字段。
