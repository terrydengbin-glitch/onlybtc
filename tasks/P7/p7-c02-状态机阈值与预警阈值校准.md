# P7-C02 状态机阈值与预警阈值校准

## 状态

DONE

## 所属 Phase

P7

## 任务目标

建立状态机阈值与预警阈值的离线校准推荐层，基于 P7-C31 BTC 4H/1D Direct Trend walk-forward 评估结果，输出可审计、可回滚、不可自动改写生产阈值的建议报告。

## 背景依据

- [开发文档.md](../../开发文档.md)
- [task index.md](../../task%20index.md)

## 实施范围

- 新增 `onlybtc.calibration.thresholds`，集中描述 P4 状态机、P4 预警、BTC Direct Trend 状态机的当前基线阈值与推荐阈值。
- 读取 `reports/btc-4h-1d-direct-trend-evaluation-report.json` 中的 walk-forward 指标、confidence calibration buckets、event window 覆盖情况。
- 对过度自信、样本不足、事件窗口覆盖缺失等情况生成阈值建议、风险说明和 rollback payload。
- 输出 `reports/p7-c02-state-alert-threshold-calibration-report.json/md`。
- 本任务只生成 recommendation，不直接修改 `p4/state_machine.py`、`p4/final_controller.py` 或 `direct_trend/state_machine.py` 的生产常量。
- 不输出交易建议，不引入开仓、止损、仓位或杠杆逻辑。

## 输入

- P7-C01 模块权重校准报告与 recommendation-only 边界。
- P7-C31 `btc_timescale_judge.v2.2` walk-forward 评估报告。
- `onlybtc.direct_trend.state_machine.DEFAULT_THRESHOLDS`。
- P4 状态机与预警层当前硬编码阈值。
- 《开发文档.md》中状态机、预警等级、数据质量和事件窗口约束。

## 输出

- 校准推荐模块：`backend/src/onlybtc/calibration/thresholds.py`。
- 生成脚本：`scripts/generate_p7_c02_state_alert_threshold_calibration.py`。
- 测试：`backend/tests/test_p7_threshold_calibration.py`。
- 报告：`reports/p7-c02-state-alert-threshold-calibration-report.json/md`。

## 验收标准

- [x] 报告明确 `applied_to_production=false`。
- [x] Direct Trend 推荐阈值从 `DEFAULT_THRESHOLDS` 派生，保留 rollback 基线。
- [x] P7-C31 过度自信信心桶能够触发更严格的 acceptance/trust/critical publish 建议。
- [x] 样本不足或事件窗口覆盖缺失时，报告必须写入 evidence gap，不得给出生产放宽建议。
- [x] 测试覆盖基线一致性、过度自信校准、缺失评估兜底和阈值边界。
- [x] 不绕过状态机、反方审查、预警等级或数据质量约束。
