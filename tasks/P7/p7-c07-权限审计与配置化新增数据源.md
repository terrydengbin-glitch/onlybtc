# P7-C07 权限、审计与配置化新增数据源

## 状态

DONE

## 所属 Phase

P7

## 任务目标

建立 Provider 权限审计与配置化新增数据源治理层，覆盖 `api_key`、`oauth/session`、`manual_login_playwright` 三类授权方式，输出脱敏 provider 权限矩阵、新增 source onboarding checklist、Glassnode 登录态审计状态和 source registry 敏感字段扫描报告。

## 背景依据

- [开发文档.md](../../开发文档.md)
- [task index.md](../../task%20index.md)

## 实施范围

- 新增 governance module 汇总 provider 权限、source registry onboarding 字段和敏感字段风险。
- Provider 级权限审计：
  - `api_key`、`oauth/session`、`manual_login_playwright` 三类授权方式。
  - 每个 provider 记录 configured、last_verified_at、permission_level、allowed_metrics。
  - 禁止在 API、日志、SQLite 明文字段中保存密钥或登录 token。
- 配置化新增数据源时必须声明：
  - 是否需要登录。
  - 是否需要付费套餐。
  - 是否允许 Playwright fallback。
  - 是否可被 P5 Settings 页面展示和测试。
- Glassnode 作为首个登录态 provider，需要支持 P1-C14 的可见性与权限审计结果。
- 输出 `reports/p7-c07-provider-permission-source-onboarding-report.json/md`。
- 本任务不读取、输出或持久化实际 API key、cookie、token、authorization header 或 storage state 内容。
- 遵守 evidence + data、历史窗口、数据质量、反证机制和预警边界。
- 不输出交易建议，不引入开仓、止损、仓位或杠杆逻辑。

## 输入

- `Settings` 中的 provider configured booleans。
- `PROVIDER_AUTH_CONFIGS` 与 `provider_auth` status。
- `SOURCE_CONFIGS` 与 source metadata。
- `.gitignore` / `.env.example` 的安全边界。
- P7-C05 Playwright stability 报告。
- P7-C06 cost/rate-limit 报告。
