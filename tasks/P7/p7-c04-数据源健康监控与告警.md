# P7-C04 数据源健康监控与告警

## 状态

DONE

## 所属 Phase

P7

## 任务目标

建立数据源健康监控与告警汇总层，复用现有 `SourceHealthEvent`、`DataQualitySnapshot`、freshness/business recency/fallback 摘要，输出可审计的 source health alert report，供 P7-C08 生产化 mock 与 Dashboard/API 后续接入。

## 背景依据

- [开发文档.md](../../开发文档.md)
- [task index.md](../../task%20index.md)

## 实施范围

- 新增治理模块，读取 SQLite 中最新数据质量快照和近期数据源健康事件。
- 按 `healthy|watch|warning|critical` 生成统一告警等级、告警原因、阻断建议和后续处理建议。
- 汇总 stale/expired/missing、business lagging、fallback、HTTP 403、registry drift、run mode mixing 等风险。
- 输出 `reports/p7-c04-source-health-monitor-report.json/md`。
- 本任务不启动采集、不改 source registry、不修改生产状态机或预警控制器。
- 遵守 evidence + data、历史窗口、数据质量、反证机制和预警边界。
- 不输出交易建议，不引入开仓、止损、仓位或杠杆逻辑。

## 输入

- `source_health_events`
- `data_quality_snapshots`
- `sources` registry metadata / freshness policy
- P7-C01/P7-C02/P7-C03 recommendation-only 治理边界
- P7-C05/P7-C06/P7-C07/P7-C08 下游任务需求

## 输出

- Source health governance module：`backend/src/onlybtc/governance/source_health.py`。
- 报告生成脚本：`scripts/generate_p7_c04_source_health_monitor_report.py`。
- 测试：`backend/tests/test_p7_source_health_monitor.py`。
- 报告：`reports/p7-c04-source-health-monitor-report.json/md`。

## 验收标准

- [x] 报告明确 `applied_to_production=false`。
- [x] 能读取最新 `DataQualitySnapshot` 并输出 source_count、score、status、freshness/business/fallback 摘要。
- [x] 近期 `SourceHealthEvent` 的 `warning|error|stale` 能触发告警项。
- [x] critical 数据质量或关键 freshness 缺口能输出 `critical`/`warning` 等级和 publish gate 建议。
- [x] 报告中明确 run mode mixing、registry drift、fallback/403 等下游处理线索。
- [x] 测试覆盖健康、warning/error、缺失快照兜底和报告生成。
- [x] 不绕过状态机、反方审查、预警等级或数据质量约束。
